UI開始導線実装メモ

## 対象範囲

初回リリースでは、開始導線を次の 3 画面相当で閉じる。

1. `main-menu`
2. `pre-generation`
3. ワールド生成後の観察状態

## 画面ごとの責務

### main-menu

- タイトルを強く見せる
- `pre-generation` への遷移ボタンを置く
- ボタン押下時に `seed` を抽選し、`PlayerPrefs` へ保存する

### pre-generation

- 保存済み `seed` を表示する
- `世界の生成` ボタンを置く
- ワールド生成と初期スポーンを開始する

### 観察状態

- `ObjectList` と `Generation` を主導線にする
- オブジェクト観察と世代更新ができる状態にする

## 初回リリースでやらないこと

- seed 手動入力
- 地形や個体数の詳細調整
- カメラ追従方式の切替 UI
- `threat map` / `heat field` の公開導線
- predator DNA export
- 詳細設定付きの世代更新 UI

## 実装上の前提

- seed 抽選と保存は `main-menu` で行う
- ワールド生成の実行は `pre-generation` で行う
- 生成後はユーザーが次に迷わないよう、観察用 UI を最小構成で見せる
