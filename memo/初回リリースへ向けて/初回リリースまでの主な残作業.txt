初回リリースまでの主な残作業

最小要件前提で、初回リリースでは「すでに動いている核を出す」ことを優先する。
そのため、`threat map` / `heat field` の公開、カメラ追従方式の切替 UI、predator DNA export、詳細設定 UI は初回リリース対象から外す。

## 初回リリースに含めるもの

- ランダム地形生成、草生成
- 草食 / 肉食の基本行動
- 炭素循環、死亡、分解
- オブジェクト一覧、対象選択、追跡、状態表示
- DNA ビューア
- 一斉世代更新
- Herbivore / Predator Genome Injector
- 動物上ゲージ表示

## 完了

### 1. 開始導線を通す

- `main-menu` と `pre-generation` のユーザー向け設計書を作成済み
- 開始導線を `main-menu` → `pre-generation` → 観察開始 の 3 段階で固定済み
- 起動直後に何を押せば観察開始できるかを文書で説明可能な状態にした

### 2. メニューを最小構成で閉じる

- ingame-menu は `ObjectList` と `Generation` の二系統に絞る方針で整理済み
- `ObjectList` で選択、追跡、状態確認まで辿る構成を文書化済み
- `Generation` で世代更新、Genome Viewer、Genome Injector まで辿る構成を文書化済み
- 初回リリース対象外のメニュー要素は中期タスク側へ退避済み

## 初回リリースまでの残作業

### 3. 初回リリース文書を最低限書く

- 起動から観察開始までの手順を書く
- オブジェクト選択、追跡、世代更新、Genome Injector の使い方を書く
- 初回リリースで含めないものを短く書く

## 初回リリースから外すもの

- `threat map` / `heat field` の公開 UI
- カメラ追従方式の切替 UI
- predator DNA export
- 世代更新の詳細設定 UI
- 動物上ゲージの表示切替 UI
- 高度な環境調整 UI
- 実験条件の保存 / 再現用 UI

## 完了条件

- 起動から観察開始までが詰まらない
- オブジェクト観察と世代更新が一通りできる
- 初回リリースで使う機能だけを文書で説明できる
