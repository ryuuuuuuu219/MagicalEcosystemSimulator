スケール設計

## 位置づけ

field / 環境場を優先した時間・空間スケールの正本。
詳細入口は `設計集約/field・環境場設計.md`。

## 時間スケール

### frame

- Unity `Update()`
- 認知、記憶更新、field debug 描画。

### physics tick

- Unity `FixedUpdate()`
- 実移動、速度。移動・待機では mana を消費しない。

### environment tick

現状は専用 tick 未実装。

- 草再生: `ResourceDispenser.Update()`
- heat 拡散: `HeatFieldManager.Update()`
- threat 減衰: `threatmap_calc.Update()`
- mana 拡散: `ManaFieldManager.Update()`

将来は field 更新周期を統一する候補がある。

### cycle

- 昼夜変化は未実装。
- Directional Light による heat field への一様入熱も未実装。

### generation

- `AdvanceGenerationController`
- 世代更新時に field を reset するかを field 種別ごとに決める。
- mana field は初期案として世代更新で 0 に戻す。

## 空間スケール

### World

- `WorldGenerator.terrainSize`
- 現在値: `512m`

用途:
- 生物の行動範囲。
- field grid の座標変換基準。

### ThreatMap

- `threatmap_calc.scoreCellSize`
- 現在値: `2m`
- `lowThreatProbeDistance = 3m`
- `lowThreatDirectionSamples = 12`

用途:
- threat score。
- 攻撃 pulse。
- 低脅威方向探索。

### HeatField

- `HeatFieldManager.gridResolution`
- 現在値: `96`
- World `512m` 基準では 1 cell 約 `5.33m`

用途:
- heat 拡散。
- 代謝熱。
- 分解熱。
- 火 / 氷属性の接続先。

### ManaField

- 未実装。
- 初期候補は HeatField と同等の world-grid 変換を持つ。
- 解像度を HeatField と揃えるか、魔法用に粗くするかは未決定。

用途:
- 魔素濃度。
- 魔法コスト。
- 空間魔法。
- 支配種進化条件。

### 属性場

- 未実装。
- 火 / 氷は heat field を使う。
- 空間は mana field を使う。
- 風は三次元 vector field として実装する。
- 雷の scalar field は設計しない。雷は着弾・連鎖・短時間 pulse の event として扱う。

### TensorField

- 未実装。
- 最初は表示・検証用。
- 実装前に scalar / vector の共通 API を整える。
