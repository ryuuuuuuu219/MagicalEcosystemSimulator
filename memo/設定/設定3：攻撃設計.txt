◇◇攻撃設計

field / 環境場の基本方針は `設計集約/field・環境場設計.md` を優先する。
攻撃は field を変化させる層であり、攻撃発生時の threat pulse、将来の属性場残留、mana 消費・生成へ接続する。

◇目的

攻撃に関する判定を、攻撃許可、攻撃選択、命中評価、被弾後イベントに分けて整理する。

このファイルは個体戦闘の共通設計とし、近接攻撃、魔法攻撃、群れ戦闘、被弾覚醒の接続点を定義する。
個別の近接攻撃形状は `設定3：近接攻撃.txt`、属性魔法は `設定5：魔法：属性の考察.txt`、被弾覚醒は `設定5：被弾による属性覚醒システム.txt` を参照する。

◇基本方針

- 草食は攻撃手段を持たない。被捕食者として回避、逃走、被弾、死亡、死骸化を担当する。
- 攻撃可能なのは肉食動物以上の相とする。
- 攻撃仕様は弧 / 接触 / 確率評価に固定する。
- 固定の攻撃距離だけで一律に停止・減速・命中させる設計は使わない。
- 攻撃は IFF 上の Hostile、または捕食対象化などの明示的な理由を持つ Neutral に対してのみ実行する。
- Friendly への攻撃は原則禁止。例外を作る場合は混乱、範囲攻撃巻き込み、指揮誤認など別システム側で理由を持たせる。

◇攻撃処理の順序

1. 対象候補の取得
- 視界、記憶、接触、群れ共有、指揮者指定から候補を得る。
- 候補には phase、speciesID、距離、方向、速度、現在の脅威度、直近の攻撃履歴を持たせる。

2. 攻撃許可判定
- `設定共通：IFF定義.txt` により Friendly / Neutral / Hostile を得る。
- Friendly は攻撃不可。
- Hostile は攻撃可能。
- Neutral は捕食対象、縄張り侵入、攻撃被害、指揮者指定などの理由がある場合のみ攻撃可能。

3. 攻撃手段の選択
- 使用可能な攻撃手段から、状況と遺伝子重みにより選択する。
- 近接 / 突進 / 噛みつき / 魔法は、最終的に同じ攻撃候補テーブルへ並べる。
- 魔法拡張後は `magicUsePriority` を、近接、追跡、回避より魔法を優先する重みとして扱う。

4. 攻撃成立評価
- 攻撃ごとの前提条件を評価する。
- 前方象限、弧、接触、エネルギー、距離確率を使う。
- 攻撃距離だけで成功を確定しない。

5. 命中・効果適用
- 命中した場合、ダメージ、ノックバック、状態異常、魔法効果を適用する。
- ダメージは攻撃種別ごとの基礎値、相対速度、属性効果で決まる。
- 範囲攻撃は巻き込み対象ごとに IFF と減衰率を評価する。

6. 攻撃イベント発火
- 攻撃トレースを生成する。
- threat map に攻撃パルスを発生させる。
- 被弾側の記憶、感情、敵対トリガー、被弾覚醒判定へイベントを渡す。

◇攻撃種別

◇突進
- 接触評価を主軸にする。
- コライダー接触により、相対速度に比例するダメージを与える。
- エネルギー消費は小。
- 命中条件は接触、前方象限、最低速度を参照する。
- 高速接触のため、攻撃後の位置ずれや地形スタック補正を考慮する。

◇噛みつき
- 接触または短い弧評価を主軸にする。
- クールダウンあり。
- 連打数に応じてクールダウンが増える。
- 成立後、自身のベクトルは相手のベクトルを参照し、引き寄せまたは追従に寄せる。
- エネルギー消費は中。

◇近接攻撃
- 弧評価を主軸にする。
- 攻撃範囲は弧を描く。
- 半径、弧の長さ、攻撃判定長さ、開始相対座標、進行ベクトルを攻撃ごとに持つ。
- クールダウンあり。
- エネルギー消費は大。

◇魔法攻撃
- 発生形式、弾道特性、属性効果に分ける。
- 詳細は `設定5：魔法：属性の考察.txt` を参照する。
- 魔法攻撃も攻撃許可判定、攻撃イベント、被弾後イベントは共通処理へ接続する。
- 被弾による属性覚醒は `設定5：被弾による属性覚醒システム.txt` 側で扱う。

◇攻撃成立評価

攻撃成立は、次の要素を組み合わせて評価する。

- 前方象限: 自身の前方に対象がいるか。
- 弧: 攻撃ごとの `AttackArcSettings` に入っているか。
- 接触: コライダーまたは接触パディング内にいるか。
- 距離確率: 距離 x に応じて攻撃試行確率 y を計算する。
- エネルギー: 攻撃コストを支払えるか。
- 対象有効性: 対象が生存し、攻撃可能関係にあるか。

現時点の近接系の距離確率案：
- 突進: `y = 80 - 2x`
- 噛みつき: `y = 60 - x`
- 近接攻撃: `y = -0.8x^2 + 70 + 7x`

この式は仮置きとし、実装では 0 から 100 の範囲へクランプする。

◇攻撃頻度

攻撃頻度は、個別周期パラメータを持たせず、クールダウンとエネルギーで制御する。

基本案：
- 攻撃ごとに固定または行動状態由来のクールダウンを持つ。
- 連打時の増幅ペナルティは必要な攻撃だけに限定する。
- 攻撃の威力は攻撃頻度から切り離し、基礎威力、相対速度、属性効果で扱う。

◇被弾後イベント

被弾時には以下を発火する。

- HP 減少
- ノックバック、鈍足、継続ダメージなどの状態変化
- 被弾者の記憶更新
- 被弾者または味方の敵対トリガー更新
- threat map への反映
- 感情パラメータへの反映
- 属性魔法による被弾覚醒判定

被弾覚醒は攻撃設計の中で直接処理せず、イベントを `設定5：被弾による属性覚醒システム.txt` へ渡す設計にする。

◇攻撃トレース・観測

攻撃は、内部判定だけでなく観測可能な痕跡を残す。

- 突進: 接触点と基準点から六方向へ展開するアスタリスク状のトレース。
- 噛みつき: 基準点周辺から X 軸へ射影する圧縮トレース。
- 近接攻撃: 弧の開始地点から終了地点へ進むトレース。
- 魔法攻撃: 属性ごとの視覚表現に従う。

攻撃トレースはデバッグ表示だけでなく、観察ゲームとしての可読性を担う。

◇未決事項

- Neutral への先制攻撃を、捕食、縄張り、資源競合のどこまで許可するか。
- Friendly への範囲攻撃巻き込みを許すか、完全に無効化するか。
- 魔法攻撃を近接攻撃と同じテーブルで抽選する際の重み正規化。
- 上位肉食動物、支配種で追加される攻撃手段をどの段階で共通化するか。
