# 設計集約

## 目的

- `memo/設定` 全体を field / 環境場優先で読み直すための入口。
- 個別メモは正本として残し、このフォルダでは参照順、責務、実装へ渡す境界を整理する。
- 設計の中心は `field・環境場設計.md` とする。

## 最優先の正本

1. `field・環境場設計.md`
- threat / heat / mana / 属性場 / テンソル場の集約。

2. `../設定1：field設計まとめ.txt`
- field 設計の詳細正本。

3. `../設定4：スケール.txt`
- 時間、距離、grid 解像度、更新周期の正本。

4. `../設定4：経済.txt`
- carbon / energy 廃止後の mana 経済正本。

5. `../設定共通：状態機械まとめ（IFF前段階）.txt`
- field を行動状態へ接続するための正本。

## 読み替えルール

- 個体行動は field を読む層。
- 戦闘と魔法は field を変化させる層。
- UI とカメラは field と対象個体を観測する層。
- 群知性と支配種は field を戦略判断へ使う層。
