群れについて

前提:
相の定義は `memo/設定/設定3：相の定義.txt` に従う。
最終的な食物連鎖ピラミッドは
草原 < 草食動物 < 肉食動物 < 上位肉食動物 < 支配種
とする。

群知性は主に肉食動物、上位肉食動物、支配種で使う。
草食動物は将来的に群れ行動を持てるが、ここでの主対象は戦術的集団行動を行う捕食相以上とする。

◇友軍判定
種族ID/speciesID が一致した個体のみ友軍とする。
この条件を満たした場合にのみ仲間として連携リストに追加する。

◇関係判定
種族ID が一致する場合は友軍とする。
それ以外は中立または敵として扱う。
群知性上の最終関係出力は「友軍」「中立」「敵」の三値とする。

肉食動物以上は同士で摂食しあうため、友軍条件から外れた個体は競争相手であり捕食対象候補でもある。
ただし競争相手は即時 Hostile ではない。縄張り侵入、資源競合、被害、指揮者指定、恨み記録などで Hostile 化する。

◇指揮
指揮力を実数値で保存し、同一群れ内で最も高い個体を指揮者とする。
指揮者は群れの `GroupIntent` と基準行動ベクトルを決める。
各個体はその基準ベクトルに、自身の行動補正を加えて移動する。

◇遺伝子候補
群れの性質は、まず指揮者側の値として扱う。
個体ごとの差分は、必要になった段階で最小限だけ追加する。

◇◇群れ性質
- 密集度（個体間距離）
- 指揮力（群れベクトルの一貫性）
- リーダー前進志向
- 包囲志向
- 追い込み志向
- 高脅威への撤退閾値
- 敵確定しやすさ
- 手加減傾向

◇◇個体差候補
- 包囲志向
- 囮耐性
- 盾抵抗値
- 側面展開志向
- リーダー追従率
- 恨み減衰率
- 執念深さ
- 臆病さ

◇リーダー行動スタイル
enum LeaderStyle
{
    FrontCommander,
    RearCommander,
    FlankCommander,
    SacrificeCommander,
    CharismaticAssault
}

先頭指揮型:
リーダーが前に出て全体を引っ張る。

後方指揮型:
リーダーは中央や後方で統率に専念する。

側面指揮型:
リーダーが横から包囲を作る。

捨て駒型:
指揮者が囮を出して自分は残る。

カリスマ突撃型:
自ら最前線へ行くので士気上昇、ただし死ぬと崩壊する。

◇高脅威目標
群れは対象ごとに高脅威スコアを計算する。
高脅威スコアがしきい値を超えた対象を、対高脅威目標として扱う。

highThreatScore =
  bodySizeScore * 1.2
+ attackRangeScore * 1.1
+ aoeScore * 1.4
+ allyDamageRateScore * 1.6
+ engagementTimeScore * 0.7
+ awakeningObservedScore * 1.5
+ selfDamageScore * 1.0
- allyCountAdvantageScore * 0.8
- decoyCandidateScore * 0.6
- escapeRouteScore * 1.0

仮しきい値:
- 3.0 以上: 警戒対象
- 5.0 以上: 高脅威対象
- 7.5 以上: 撤退・散開候補

◇高脅威スコアの仮重み

| 評価軸 | 値の意味 | 仮重み |
| --- | --- | --- |
| bodySizeScore | 相手の体格差 | 1.2 |
| attackRangeScore | 攻撃範囲の広さ | 1.1 |
| aoeScore | AoEの有無・広さ | 1.4 |
| allyDamageRateScore | 味方被害速度 | 1.6 |
| engagementTimeScore | 交戦時間 | 0.7 |
| awakeningObservedScore | 被弾覚醒の観測 | 1.5 |
| selfDamageScore | 自身の受傷量 | 1.0 |
| allyCountAdvantageScore | 味方数による優位 | -0.8 |
| decoyCandidateScore | 囮候補数 | -0.6 |
| escapeRouteScore | 退路の有無 | -1.0 |

補足:
- 正の重みは脅威を上げる。
- 負の重みは対処可能性を表し、脅威を下げる。
- 各スコアは原則 0.0 - 1.0 に正規化する。

◇対高脅威目標への出力
- 高脅威スコアが低い: 通常の Attack / Encircle を許可。
- 警戒対象: Encircle / Drive を優先。
- 高脅威対象: Lure / Retreat / ShieldLeader を候補に入れる。
- 撤退・散開候補: Retreat / Scatter を優先。

◇群れ側の出力
enum GroupIntent
{
    Attack,
    Encircle,
    Drive,
    Lure,
    Retreat,
    Scatter,
    ShieldLeader
}

Attack:
戦う。

Encircle:
囲んで様子を見る。

Drive:
追い込む。

Lure:
誘導する。

Retreat:
逃げる。

Scatter:
散開する。

ShieldLeader:
盾役を前に出す。

◇目標識別レベル
enum TargetStatus
{
    Unknown,   // 未観測
    Bogey,     // 未確認目標
    Suspect,   // 敵性疑い
    Neutral,   // 中立
    Hostile,   // 敵
    Friendly   // 味方
}

◇IFF（Identification Friend or Foe）
「speciesID」種族ID
「phase」相

if (target.speciesID == self.speciesID)
    status = Friendly;

if (hostility trigger is active)
    status = Hostile;
else
    status = Neutral;

補足:
Unknown、Bogey、Suspect は観測や識別途中の内部状態としては残してよい。
最終的な群れ行動の意思決定では Friendly / Neutral / Hostile に収束させる。
phase は捕食段階の参照用であり、Friendly 判定そのものは speciesID の一致で決める。

◇敵確定の条件
Hostile 判定は次のどれかで起こす。

- 攻撃された
- 味方が攻撃された
- 相と群知性条件に基づく敵対
- 距離識別
- 以前の恨み記録あり
- 指揮者が敵指定した

◇基本ルール
- speciesID same なら Friendly
- それ以外は Neutral を基本とし、敵対トリガー成立時に Hostile
- 指揮者は GroupIntent と基準ベクトルを出す
- 個体は自身の状態に応じて基準ベクトルを補正する

◇相との接続
草食動物は主に被食相として群れ防衛や逃走共有に使う。
肉食動物は草食動物を主食とするが、友軍条件から外れた肉食動物以上も敵対対象になる。
上位肉食動物は下位相捕食と同格戦闘の両方を前提にする。
支配種は最上位相として、敵対群れの制圧、資源帯の支配まで含めて扱う。
