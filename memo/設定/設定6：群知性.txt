群れについて

前提：
相の定義は `memo/設定/設定3：相の定義.txt` に従う。
最終的な食物連鎖ピラミッドは
草原 < 草食動物 < 肉食動物 < 上位肉食動物 < 支配種
とする。

群知性は主に肉食動物、上位肉食動物、支配種で使う。
草食動物は将来的に群れ行動を持てるが、ここでの主対象は戦術的集団行動を行う捕食相以上とする。

◇友軍判定
種族ID/speciesID と 派閥ID/factionID の両方が一致した個体のみ友軍とする。
この条件を満たした場合にのみ仲間として連携リストに追加する。

◇関係判定
種族ID と派閥ID の両方が一致する場合は友軍とする。
それ以外は中立または敵として扱う。
群知性上の最終関係出力は「友軍」「中立」「敵」の三値とする。

これは特に肉食動物以上に作用し、IFF の根幹ルールになる。
肉食動物以上は同士で摂食しあうため、友軍条件から外れた個体は競争相手であり捕食対象候補でもある。
ただし競争相手は即時 Hostile ではない。縄張り侵入、資源競合、被害、指揮者指定、恨み記録などで Hostile 化する。

◇IFF（Identification Friend or Foe） 
「speciesID」種族ID
「factionID」派閥ID
「phase」相

if (target.speciesID == self.speciesID && target.factionID == self.factionID)
    status = Friendly;

if (hostility trigger is active)
    status = Hostile;
else
    status = Neutral;

補足：
Unknown、Bogey、Suspect は観測や識別途中の内部状態としては残してよい。
最終的な群れ行動の意思決定では Friendly / Neutral / Hostile に収束させる。
phase は捕食段階の参照用であり、Friendly 判定そのものは speciesID と factionID の一致で決める。

◇敵確定の条件
Hostile判定は次のどれかで起こします。

・攻撃された
・味方が攻撃された
・相と群知性条件に基づく敵対
・距離識別
・以前の恨み記録あり
・指揮者が敵指定した

◇やってみたいこと
執念深さが忠誠度を上回り命令不服従を起こす
指揮者が過去に恨みを蓄積させた個体に向かって一斉攻撃を命ずる展開


◇基本ルール
speciesID same かつ factionID same なら Friendly
それ以外は Neutral を基本とし、敵対トリガー成立時に Hostile
commandPower が近い個体同士は派閥候補
loyalty が低い個体は追従率低下
rebelliousness が高い個体は命令を部分拒否

◇相との接続
草食動物は主に被食相として群れ防衛や逃走共有に使う。
肉食動物は草食動物を主食とするが、友軍条件から外れた肉食動物以上も敵対対象になる。
上位肉食動物は下位相捕食と同格戦闘の両方を前提にする。
支配種は最上位相として、敵対群れの制圧、森を使った奇襲、資源帯の支配まで含めて扱う。

◇森の扱い
森は視界遮蔽を持つ地形であり、群れの誘導、伏撃、退避、包囲崩しに使う。
また森の周囲には草原が生成される前提のため、群れは森縁を資源帯と遮蔽帯が接続する重要ラインとして認識する。

◇派閥形成条件の例
指揮者との commandPower 差が小さい
loyalty が低い
目標評価が食い違う
味方被害が増えている
指揮者が自分を盾に使った
