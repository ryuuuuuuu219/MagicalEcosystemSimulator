# 設定

## 読み方

このフォルダの設計は、まず field / 環境場を中心に読む。
個体 AI、戦闘、魔法、群れ、支配種は、それぞれ環境場を参照・変化・可視化する層として扱う。

最初に読む正本:

1. `設計集約/field・環境場設計.md`
2. `設定1：field設計まとめ.txt`
3. `設定4：スケール.txt`
4. `設定4：経済.txt`
5. `設定共通：状態機械まとめ（IFF前段階）.txt`

## 設計の優先順位

1. field / 環境場
- threat map、heat field、mana field、属性場、テンソル場。
- 場の値、勾配、更新周期、可視化を先に決める。

2. 個体 AI
- 個体は field を参照して移動・回避・追跡・離脱を決める。
- 行動状態は field の値だけで決めず、視認、距離、記憶、境界、mana 状態も合わせて判断する。

3. 戦闘
- 攻撃は field に影響を残す。
- 攻撃トレース、threat pulse、ダメージ、目標失効理由を観測できるようにする。

4. 魔法
- 魔法は mana field と属性場を消費・生成・変形する行動として扱う。
- 火と氷は heat field、空間は mana field、風は三次元 vector field と接続する。
- 雷の scalar field は設計しない。雷は着弾イベント、連鎖イベント、短時間 pulse として扱う。

5. 群れ・支配種・課題ステージ
- 上位タスクは field を戦略判断・環境支配・検証条件として使う。

## 実装へ渡す原則

- field の生成処理を重複させない。
- `ThreatMapsGenerator` と `HeatFieldManager` は既存基盤として扱う。
- mana field は実装済みの基盤として扱う。次は個体吸収、魔法消費、相進化判定へ接続する。
- UI 表示と field 更新を同じクラスに詰め込まない。
- 戦闘可視化は `memo/タスク一覧/1.戦闘可視化強化`、移動判断は `memo/タスク一覧/2.生態系コア強化`、魔素基盤は `memo/タスク一覧/5.魔素基盤構築` を入口にする。
