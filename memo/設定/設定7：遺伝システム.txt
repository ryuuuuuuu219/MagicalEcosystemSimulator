遺伝システム

◇初期値　
既存ゲノム優先、固定値+乱数（レンジ固定）

◇世代交代時　
候補をスコア順に並べて上位2体を親にするか、Selection モードでは保存済みDNAプールからランダムに2つ取ります。
交叉方法は BlendFloat で切り替えています。
Assign: 親A/Bどちらかをそのまま採用
Average: 平均
Interpolate: 親A-B間をランダム補間

◇突然変異　
GlobalRange モードなら、globalMutationMin から globalMutationMax の加算値を入れます。
それ以外は、親2体の平均絶対値を基準に parentMutationScale 倍の範囲で増減します。
つまり「割合変異」はあります。
ただし「この項目は最大 ±3、この項目は ±0.2」みたいな個別上限指定にはなっていません。

◇

いまの突然変異の増減値は、各項目ごとに個別設定ではなく、全項目共通ルールです。AdvanceGenerationController.cs:63

共通設定の初期値

enableMutation = true
mutationChance = 0.1
mutationRangeMode = ParentRelative
globalMutationMin = -0.2
globalMutationMax = 0.2
parentMutationScale = 0.2
増減の決まり方

まず各項目ごとに 10% の確率で突然変異します。AdvanceGenerationController.cs:1239
変異しなければ増減 0
変異したときの増減幅はモード次第です
1. GlobalRange モード

増減値 = -0.2 から +0.2 の範囲の一様乱数
式: value + random(globalMutationMin, globalMutationMax)
つまり最大変化量は固定で ±0.2 です。AdvanceGenerationController.cs:1242
2. ParentRelative モード

現在のデフォルトはこれです
親2体の平均絶対値を基準にして、その 20% 分だけ上下します
式:
parentBase = max(0.01, abs((parentA + parentB) / 2))
delta = random(-parentBase * 0.2, parentBase * 0.2)
value += delta
つまり最大変化量は 親平均の絶対値 × 0.2 です。AdvanceGenerationController.cs:1248
具体例

親平均が 10 の項目
変異幅は -2.0 から +2.0
親平均が 5 の項目
-1.0 から +1.0
親平均が 1 の項目
-0.2 から +0.2
親平均が 0 付近の項目
下限補正で parentBase = 0.01
-0.002 から +0.002
対象

草食・捕食どちらも、ほぼ全ての float 項目にこのルールがかかります
Vector3 や WaveGene も中の各 float を同じルールで変異させています。AdvanceGenerationController.cs:1258

◇◇
追加要件
すべての乱数を２回出力→平均にして中央値によるようにする
補足：これの結果は正規分布ではなく三角形の形で分布する

◇◇
前提
草食動物は純粋な被捕食者として定義し、進化しない。
進化の初期値は肉食動物とし、HPの駆け引きは肉食動物、上位肉食動物、支配種でおこなう。
肉食動物以上の相では、友軍以外が Neutral または Hostile になり、敵対トリガー成立時に攻撃・被弾する設計を予定。

補足:
ここでいう「草食動物は進化しない」は相の進化を指す。
organ の獲得、欠落、level 変化は草食動物も対象にできる。
ただし `PredatorPhaseEvolutionAction` による phase 昇格判定は肉食動物以上に限定する。

◇◇
speciesID 付与ルール

phase 昇格時に speciesID を確率で付与・更新する。
候補は既存IDに加えて、既存最大ID + 1 までを含める。
抽選重みは 0 に近いIDほど大きく、大きいIDほど減衰するテーブルを使う。
新IDを含めることで新種分岐を許容しつつ、低IDの既存種が多数派として残りやすくする。
