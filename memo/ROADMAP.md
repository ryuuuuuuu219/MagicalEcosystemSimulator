# ROADMAP

このファイルは、`memo/` 配下のファイル構成と運用ルールを説明する。

## 基本方針

- 基準READMEは `memo/README.md` を使用する。
- ユーザー向け文書は `document/` に置く。
- 開発者向けメモ・設計・タスクは `memo/` に置く。
- `memo/チェンジログ.txt` は不使用とし、履歴は各タスク文書またはコミット履歴で管理する。

## ディレクトリ構成

### `memo/タスク一覧/`

- 実作業タスクを番号付きで管理する。
- 既存資産フォルダ:
  - `0.遺伝子設計変更/`
- 今後の主要タスクフォルダ:
  - `1.生態系コア強化/`
  - `2.挙動AI分離/`
  - `3.戦闘システム拡張/`
  - `4.魔素基盤構築/`
  - `5.属性魔法拡張/`
  - `6.群知性導入/`
  - `7.学習と進化/`
  - `8.支配種到達条件/`
- 各タスクフォルダ内の `README.md` に目的・対象・参照を記述する。
- 補助資料:
  - `0.遺伝子設計変更/現状の遺伝子構造まとめ.txt`
  - `0.遺伝子設計変更/メニュー整備予定(長期目標).txt`

### `memo/タスク一覧/` 内の設定統合ファイル

- 設定系テキストは各作業フォルダへ `設定：*.txt` として統合管理する。
- 例: `9.課題ステージ/設定：ゲーム設計.txt`、`10.課題ステージ（期間イベント_環境外乱つき）/設定：環境外乱.txt` など。

## 更新手順

1. 新しい作業は `memo/タスク一覧/n.○○/README.md` に追加する。
2. 仕様変更がある場合は各作業フォルダ内の `設定：*.txt` を更新する。
3. 全体方針や参照先変更がある場合のみ `memo/README.md` とこのファイルを更新する。
