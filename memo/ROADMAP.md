# ROADMAP

この文書は `memo/` 配下の各メモを整理し、実装順と設計課題を一本化したロードマップです。
実装済みの基準は [README.md](/c:/魔法環境シミュ/README.md) を優先し、詳細な論点は各メモを参照します。

## 方針

1. 先に設計の共通前提をそろえる。
2. 次に生態系コアを強化し、観察可能な差を増やす。
3. その上で戦闘、群知性、魔法、支配種へ広げる。
4. UI は独立機能ではなく、観測と調整の基盤として並行整備する。

## 現状の整理

### 実装済み、または土台があるもの

- 草食・肉食の基本行動ループ
- threat map の計算と debug 可視化
- heat field の計算と debug 可視化
- 遺伝子閲覧、DNA 入力、世代更新 UI
- オブジェクト一覧、注目、状態表示 UI
- 動物上ゲージの描画基盤

### 未実装、または設計止まりのもの

- mana field / 魔素場
- 群れ意思決定と `GroupIntent`
- 種族 ID / 派閥 ID / IFF
- biome の体系化
- 属性魔法
- 支配種到達条件
- 統合された環境調整 UI

## Phase 0: 設計基準の固定

### 目的

- 以降の実装で迷わないための単位、更新責務、評価基準を固定する。
- 既存実装にある暗黙仕様を文書へ引き上げる。

### 主な対象

- 時間スケール
- 空間スケール
- AI 更新モデル
- 行動決定アルゴリズム
- 感覚モデル
- 進化評価式
- 環境場の責務分離

### 現状

- `Update` と `FixedUpdate` の責務分離は実装に存在するが、統一文書化は不十分。
- `threatmap_calc`、`HeatFieldManager`、`WorldGenerator` に個別スケールが存在する。
- 草食・肉食の行動合成は実装済みだが、共通仕様としての整理が不足している。
- mana 系は未着手。

- threat map は部品実装済みだが、主移動ロジックへの全面統合は未完。
