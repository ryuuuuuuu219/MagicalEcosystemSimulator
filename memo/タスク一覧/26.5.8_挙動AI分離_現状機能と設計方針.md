# 26.5.8 挙動AI分離 現状機能と設計方針

作成日: 2026-05-08

## 目的

`herbivoreBehaviour` / `predatorBehaviour` に集中している感覚、欲求、移動、攻撃、環境反応を、将来的に `AddComponent<FoodVisionSense>()` のような独立した挙動コンポーネントとして扱える形へ分離する。

最終的には、各機能を「存在するか」と「どの程度強いか」で扱い、遺伝的アルゴリズムが環境に応じて機能を取捨選択できるようにする。

```text
Behaviour AI = Brain + Sense Components + Desire Components + Action Components
Gene target = component enabled + component level + local tuning values
```

## 現状の主要スクリプト

- `Assets/script/Ingame/Creatures/Herbivore/herbivoreBehaviour.cs`
  - 草食個体の生命、mana、視覚、記憶、捕食者回避、餌移動、摂食、移動合成を持つ。
- `Assets/script/Ingame/Creatures/Predator/predatorBehaviour.cs`
  - 捕食個体の生命、mana、獲物探索、脅威探索、比例航法、攻撃、死体摂食、脅威パルス、phase evolution を持つ。
- `Assets/script/Ingame/AI/AnimalAICommon.cs`
  - 記憶更新、脅威度計算、波形値、境界回避、地形追従移動などの共通関数。
- `Assets/script/Ingame/Combat/PredatorCombatLibrary.cs`
  - 突進、噛みつき、近接攻撃の判定とダメージ計算。
- `Assets/script/Ingame/Creatures/Herbivore/HerbivoreGenome.cs`
  - 草食AIの移動、視覚、摂食、恐怖、回避、徘徊パラメータ。
- `Assets/script/Ingame/Creatures/Predator/PredatorGenome.cs`
  - 捕食AIの移動、視覚、追跡、攻撃、脅威、戦闘演出パラメータ。

## 分離後の基本構造

設計後の機能一覧は `26.5.8_挙動AI分離_機能一覧_設計後.md` に分離する。

### 1. Brain

`AnimalBrain` は、個体の中核として各コンポーネントの結果を集める。

責務:

- 個体共通の `Context` を保持する。
- Sense を tick して記憶と認識を更新する。
- Desire から移動候補を集める。
- Action を実行する。
- 最終的な移動ベクトルを `GroundMotor` に渡す。

Brain 自体は「餌とは何か」「攻撃とは何か」をなるべく知らない。

### 2. Context

コンポーネント間で共有する最小情報。

候補:

- `GameObject Self`
- `Transform Transform`
- `Terrain Terrain`
- `Resource BodyResource`
- `float Health`
- `float Mana`
- `float MaxMana`
- `Vector3 CurrentVelocity`
- `AIMemoryStore Memory`
- `CreatureRelationResolver RelationResolver`
- `ResourceDispenser ResourceDispenser`

まずは既存 `herbivoreBehaviour` / `predatorBehaviour` から参照を渡す薄い構造体で始め、後で MonoBehaviour 化する。

### 3. Sense

外界を検知し、記憶や認識キャッシュを更新する。

インターフェイス候補:

```csharp
public interface IAISense
{
    void TickSense(AIContext context, float deltaTime);
}
```

例:

- `FoodVisionSense`
- `PredatorVisionSense`
- `PreyVisionSense`
- `ThreatMapSense`
- `ManaFieldSense`

### 4. Desire

移動したい方向と重みを返す。

インターフェイス候補:

```csharp
public interface IAIDesire
{
    AIMoveIntent Evaluate(AIContext context);
}
```

```csharp
public struct AIMoveIntent
{
    public Vector3 direction;
    public float weight;
    public string reason;
}
```

例:

- `FoodDesire`
- `PreyChaseDesire`
- `ThreatAvoidanceDesire`
- `BoundaryAvoidanceDesire`
- `WanderDesire`
- `ManaFieldAttractionDesire`

### 5. Action

副作用を持つ処理。mana 増減、ダメージ、攻撃、摂食、脅威パルスなど。

インターフェイス候補:

```csharp
public interface IAIAction
{
    bool TryAct(AIContext context, float deltaTime);
}
```

例:

- `GrassEatAction`
- `CorpseEatAction`
- `ChargeAttackAction`
- `BiteAttackAction`
- `MeleeAttackAction`
- `FieldManaAbsorbAction`
- `ThreatPulseEmitter`
- `PredatorPhaseEvolutionAction`

## 遺伝子との接続方針

現状の `HerbivoreGenome` / `PredatorGenome` は「機能が必ず存在する前提のパラメータ集」になっている。

分離後は、機能ごとに `enabled` と `level` を持たせる。

```csharp
[System.Serializable]
public struct AIComponentGene
{
    public string componentId;
    public bool enabled;
    public float level;
}
```

最初から既存 genome を全部置き換えない。移行期は以下の形にする。

1. 既存 genome は維持する。
2. 新しい component gene は追加フィールドとして持つ。
3. component gene が空なら、現状と同じ挙動になる default set を生成する。
4. 各コンポーネントは既存 genome の値を読みながら、徐々に自分専用の gene 値へ移す。

## 推奨する実装順

### Phase 1: 壊さない足場作り

- `AIContext`
- `AIMoveIntent`
- `IAISense`
- `IAIDesire`
- `IAIAction`
- `AnimalBrain`

この段階では既存挙動はまだ `herbivoreBehaviour` / `predatorBehaviour` が保持する。

### Phase 2: 共通移動を外へ出す

- `GroundMotor`
- `BoundaryAvoidanceDesire`
- `WanderDesire`

移動結果が変わるリスクが比較的小さいものから移す。

### Phase 3: 草食の感覚と欲求を分ける

- `FoodVisionSense`
- `PredatorVisionSense`
- `FoodMemory`
- `ThreatMemory`
- `FoodDesire`
- `ThreatAvoidanceDesire`
- `GrassEatAction`

草食は攻撃が少ないため、最初の分離対象に向く。

### Phase 4: 捕食の追跡と攻撃を分ける

- `PreyVisionSense`
- `PreyChaseDesire`
- `ProportionalNavigationSteering`
- `CorpseEatAction`
- `ChargeAttackAction`
- `BiteAttackAction`
- `MeleeAttackAction`

`PredatorCombatLibrary` はすでに分離済みなので、Action から呼ぶ形へ寄せる。

### Phase 5: 遺伝的ON/OFFへ接続

- `AIComponentGene`
- `AIComponentSet`
- `AnimalAIInstaller`

manager から生成時に component set を配布し、必要な MonoBehaviour を AddComponent する。

## 直近の注意点

- `herbivoreBehaviour` と `predatorBehaviour` は Unity scene / prefab から直接参照されている可能性が高いため、最初から削除しない。
- まずは既存クラスを facade として残し、中身だけ新コンポーネントへ委譲する。
- `HerbivoreGenome` / `PredatorGenome` の serialized field は scene に保存されている可能性があるため、急に削除しない。
- `ResourceDispenser` や UI は既存 behaviour 型を見ている箇所があるため、外部 API は段階的に維持する。
- 乱数や波形を含むため、分離後は完全一致よりも「餌へ向かう、逃げる、攻撃する」という行動カテゴリが維持されているかを見る。

## 最初に切り出す候補

最小リスクで始めるなら、以下の順がよい。

1. `BoundaryAvoidanceDesire`
2. `WanderDesire`
3. `FieldManaAbsorbAction`
4. `ThreatPulseEmitter`
5. `GrassEatAction`

これらは対象探索や戦闘判定より依存が少なく、分離後の土台を検証しやすい。
