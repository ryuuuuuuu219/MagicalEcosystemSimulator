# 26.5.8 organ 設計

作成日: 2026-05-08

## 目的

organ の定義、導入確率、依存関係、旧 behaviour 維持方針をまとめる。

現時点のコードでは、`AnimalAIInstaller.InstallDefaultOrgans()` が共通基盤を導入し、`OrganPresetLibrary` が草食・捕食者・上位捕食者ごとの固定構成を `Ensure()` で配布する。

```text
AnimalBrain
AIMemoryStore
GroundMotor
CreatureMotorBootstrap
```

このフォルダでは、今後 `AIComponentGene.enabled` を生成するときの基準確率と、相進化時に `Ensure()` される固定構成を整理する。

## ファイル

- `organ定義.md`
  - organ の定義と、ここまでの読み取り事項。
- `コンポーネント導入確率表.md`
  - コンポーネントごとの導入確率表。
- `インストーラー確率方針.md`
  - 確率設計のルール、固定導入、自動導入、species別 default set の考え方。
- `organ依存関係.md`
  - organ の自動導入・前提関係。
- `旧behaviour維持設計.md`
  - organ 化後の `herbivoreBehaviour` / `predatorBehaviour` の維持・Ensure 設計。
- `メモ上の矛盾.txt`
  - 解決済みを除いた残課題。

## 前提

- 確率は初期案。`OrganPresetLibrary` の固定構成を土台に、実装後のシミュレーション結果で調整する。
- Core / Motor / Memory の一部は生存に必要なので、確率抽選ではなく固定導入にする。
- `ThreatPulseEmitter` は gene の直接対象ではなく、攻撃 organ が入った場合に自動導入する。
- `ZigzagEvasionSteering` は廃止予定なので対象外。
