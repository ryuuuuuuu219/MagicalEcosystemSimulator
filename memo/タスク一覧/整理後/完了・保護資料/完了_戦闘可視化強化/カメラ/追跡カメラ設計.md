# 追跡カメラ設計

## 目的

- 指定個体と目標個体を、ポケモン風の見やすい戦闘構図で表示する。
- 指定個体を画面左下、目標個体を画面右上へ配置する。
- カメラの Z 軸回転は使わず、カメラ位置と向きだけで構図を作る。

## 対象オブジェクト

- 指定個体 `subject`
- 目標個体 `target`
- カメラ `camera`

## 条件

- カメラは指定個体から一定距離 `cameraDistance` に配置する。
- `camera.transform.rotation.z` は使わない。
- 画面上で指定個体は左下、目標個体は右上に配置する。
- `∠ subject-camera-target` の相対角度は camera FOV の `0.7f` にする。
- 候補点が複数ある場合は、より高い点を優先する。

## 幾何イメージ

指定個体を中心とした半径 `cameraDistance` の球面上にカメラ候補を置く。

```text
|camera - subject| = cameraDistance
```

さらに、カメラから見た指定個体と目標個体の見かけ角を、FOV の `0.7f` に近づける。

```text
angle(subject - camera, target - camera) ~= camera.fieldOfView * 0.7f
```

この条件を満たす点は球面上に円状に分布しうる。
その中から、指定個体から目標個体へのベクトルを基準に「右へ 135 度、上へ 45 度」寄りの候補を選ぶ。

## 基準ベクトル

```text
toTarget = normalize(target.position - subject.position)
up = Vector3.up
right = normalize(cross(up, toTarget))
```

`toTarget` がほぼ垂直の場合は、カメラの現在 right か world right を fallback にする。

## 初期候補

指定個体から見て、目標方向の反対寄りかつ右上側へカメラを置く。

```text
horizontal = rotateAroundUp(toTarget, -135deg)
candidateDir = normalize(horizontal + up * tan(45deg))
cameraPos = subject.position + candidateDir * cameraDistance
```

この候補は「指定個体から目標へのベクトルを右に 135 度、上に 45 度した直線」と、指定個体中心の球の交点として扱う。

## 見かけ角の補正

初期候補で見かけ角が `FOV * 0.7f` から外れる場合、候補を球面上で調整する。

調整方針:

1. `subject` 中心の球面上に保つ。
2. Z 軸回転なしで `subject` が左下、`target` が右上に見える向きを保つ。
3. 目標角に近い候補を選ぶ。
4. 同程度なら高い候補を選ぶ。

実装候補:

```csharp
float targetAngle = camera.fieldOfView * 0.7f;
Vector3 bestPos = initialCandidate;
float bestScore = float.PositiveInfinity;

for each yawOffset around preferred direction:
    for each pitchOffset around preferred height:
        pos = subject.position + dir * cameraDistance;
        visibleAngle = Vector3.Angle(subject.position - pos, target.position - pos);
        compositionScore = EvaluateScreenPlacement(pos);
        score = abs(visibleAngle - targetAngle) + compositionScore;
        prefer higher pos.y when scores are close;
```

## カメラ向き

カメラは指定個体と目標個体の中間より少し指定個体寄りを見る。

```text
lookPoint = lerp(subject.position, target.position, 0.45f)
```

Z 軸回転を避けるため、`LookRotation(forward, Vector3.up)` を使う。

```csharp
Vector3 forward = normalize(lookPoint - cameraPos);
camera.rotation = Quaternion.LookRotation(forward, Vector3.up);
```

## 画面配置判定

`WorldToViewportPoint` で構図を確認する。

目標:

```text
subject viewport x < 0.5
subject viewport y < 0.5
target viewport x > 0.5
target viewport y > 0.5
```

理想値:

```text
subject ~= (0.30, 0.30)
target  ~= (0.70, 0.70)
```

評価例:

```text
compositionScore =
  distance(subjectViewport.xy, (0.30, 0.30)) +
  distance(targetViewport.xy,  (0.70, 0.70))
```

## フォールバック

- 目標個体がない場合は指定個体追尾カメラへ戻す。
- 指定個体と目標個体が近すぎる場合は、見かけ角条件より画面配置を優先する。
- 地形にめり込む場合は上方向へ押し上げる。
- カメラと対象の間に地形が入る場合は、距離を縮めるか高さを上げる。
