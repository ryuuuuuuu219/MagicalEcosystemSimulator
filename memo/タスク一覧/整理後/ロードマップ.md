# 整理後タスク ロードマップ

`0.phase0_organ設計` から `3.世代更新と遺伝` までの責務を分離し、organ foundation、organ 継承、organ 突然変異、世代更新を混ぜないための進行表。

## 基本方針

- `0.phase0_organ設計` は固定 organ foundation の確立に集中する。
- `2.organ構成の遺伝` は親から子へ organ 構成が伝わる仕組みに集中する。
- `2.5.organ突然変異・生存中可塑性` は organ の着脱、重み変化、生存中変化を扱う。
- `3.世代更新と遺伝` は評価、選抜、交叉、突然変異呼び出し、ログ、DNA 入出力を担当する。
- 生存中 mutation は `OrganFoundation` checkpoint として保存し、世代更新時の評価候補に含める。
- 生存中 mutation で獲得した organ / 魔法資質も、評価を通過した場合は次世代へ遺伝可能にする。
- 生存中 mutation は `OrganFoundation` の 10 秒 counter と `AIComponentGene.mutationChanceT` で抽選する。
- 世代更新時 mutation は `AIComponentGene.mutationChanceG` で抽選する。
- magic gene は分離せず、魔法資質は organ gene に含める。

## 0. phase0_organ設計

目的は、旧 behaviour 本流から organ foundation 本流へ移行できる足場を固定すること。

### 担当

- `OrganFoundation` が `AnimalBrain.TickBrain()` を呼ぶ。
- 旧 `herbivoreBehaviour` / `predatorBehaviour` と二重移動・二重攻撃を起こさない。
- `OrganRelationLibrary` が依存 organ を自動導入する。
- `OrganPresetLibrary` は phase / species ごとの固定初期構成を返す準備をする。
- `AIComponentGene` は最小データ構造として扱い、突然変異ロジックは持たせない。
- `AIComponentGene.isVitalOrgan` の概念は持たせるが、mutation 実行ロジックは持たせない。
- `AIComponentGene.isVestigialOrgan`、`mutationChanceT`、`mutationChanceG` の値は持たせる。

### 完了条件

- 固定 preset から期待 organ set が再現できる。
- `AnimalBrain` が本流 runner として動く。
- 旧 behaviour は互換 API、状態保持、UI / manager 接続に退いている。

## 1. 支配種までの遺伝子定義

目的は、草食・捕食・上位捕食・支配種までの genome 項目と境界を定義すること。

### 担当

- 草食 genome と捕食 genome の対応表。
- phase / speciesID / dominant 到達に必要な値。
- organ gene を genome のどこに持つか。
- UI / DNA 表示対象と内部専用値の分離。

### 完了条件

- organ 構成、相進化、魔法資質、世代更新が参照する genome 境界が分かる。

## 2. organ構成の遺伝

目的は、親から子へ organ 構成が継承される仕組みを作ること。

### 担当

- `AIComponentSet` を genome または DNA 表現へ接続する。
- crossover で `AIComponentGene` を継承する。
- phase / species preset と genome 差分を合成する。
- required organ は必ず保持し、optional organ は個体差として扱う。

### 完了条件

- 親の organ 構成差が次世代に再現可能に伝わる。
- mutation なしでも個体差 organ の継承経路が追える。

## 2.5. organ突然変異・生存中可塑性

目的は、organ 構成の変化を `2.organ構成の遺伝` と `3.世代更新と遺伝` から分離すること。

### 担当

- 世代更新時 mutation による `AIComponentGene.enabled` / `level` の変化。
- 生存中の低確率 mutation による runtime organ 構成変化。
- organ 着脱、または安全な無効化。
- organ 着脱時の `OrganFoundation` checkpoint 保存。
- 相進化時と organ 変化時の snapshot 保存。
- 痕跡器官 `isVestigialOrgan` と `VestigialOrgans` の管理。
- mutation 後の依存 organ 再解決。
- required organ の保護。
- mutation log と generation log への接続点。

### 注意

- 生存中 mutation は `OrganFoundation` checkpoint として保存する。
- checkpoint は世代更新時に評価候補へ含め、選抜を通過した場合は次世代へ遺伝可能にする。
- checkpoint は相進化時と organ 変化時に snapshot として保存する。
- `AnimalBrain` 実行中の Add / Remove 後は `RefreshOrgans()` が必要。
- 初期実装では RemoveComponent より `enabled=false` / `gene.enabled=false` / `weight == 0` による早期 return を優先する。
- 失われた organ は削除ではなく `isVestigialOrgan` と `VestigialOrgans` で痕跡器官として保持する。
- `AIComponentGene.isVitalOrgan` が true の organ は mutation による着脱対象から除外する。
- parent organ を無効化した場合でも、依存 organ は他の有効 organ から参照されていないときだけ無効化候補にする。

### 完了条件

- required organ が mutation で外れない。
- optional organ の enabled / level が確率的に変化する。
- 生存中 mutation が 10 秒間隔で抽選される。
- 失われた organ が痕跡器官として追跡できる。
- mutation 後も依存 organ が欠けない。
- 生存中 mutation の checkpoint が世代更新時評価候補に含まれる。
- 相進化時と organ 変化時の snapshot が残る。
- 生存中 mutation で獲得した organ / 魔法資質が、評価後に次世代へ遺伝可能である。
- 生存中 mutation と世代更新時 mutation のログが区別できる。

## 3. 世代更新と遺伝

目的は、評価、選抜、交叉、突然変異呼び出し、再配置、ログ、DNA 入出力をまとめること。

### 担当

- fitness 評価。
- parent selection。
- crossover。
- `2.5.organ突然変異・生存中可塑性` の世代更新時 mutation 呼び出し。
- 生存中 mutation checkpoint の評価候補化。
- generation log と DNA validation。
- organ gene 表示。

### 完了条件

- 世代更新の入出力が README から追える。
- organ gene の継承と mutation がログで確認できる。
- 草食・捕食の genome 形式差による表示・注入事故が起きない。

## 問題なしと判断できる点

- Phase0 から突然変異を外すことで、foundation の責務が軽くなる。
- `2.organ構成の遺伝` と `2.5.organ突然変異・生存中可塑性` を分けることで、継承と変異を別々に検証できる。
- `3.世代更新と遺伝` は mutation ルール本体を持たず、呼び出しとログに集中できる。

## 残るリスク

- 生存中 mutation の即時 Remove は Unity の `Destroy()` タイミングと `AnimalBrain` のキャッシュに注意が必要。初期実装では早期 return 型の無効化を優先する。
- 依存 organ の自動導入だけでなく、親 organ 無効化時の reverse dependency 判定が必要。
- checkpoint と genome 確定値の境界を UI / log に出さないと、観測時に混同しやすい。
- required / optional の分類が曖昧なままだと、mutation で個体の基盤が壊れる。
