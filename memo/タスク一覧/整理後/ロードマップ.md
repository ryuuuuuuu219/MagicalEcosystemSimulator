# 整理後タスク ロードマップ

`0.phase0_organ設計` から `5.支配種仕様・役割定義` までの責務を分離し、genome 正本、organ 継承、organ 突然変異、世代更新、相進化、支配種仕様を混ぜないための進行表。

## 現在の到達点

- phase0 の organ foundation は実装済み。`OrganFoundation.Update()` が `AnimalBrain.TickBrain()` を呼ぶ。
- `OrganPresetLibrary` は固定 Ensure 中心から `AIComponentSet` preset 生成へ移行済み。
- `AIComponentGene` / `AIComponentSet` は enabled、level、weight、vital、vestigial、mutation chance を持つ。
- `OrganRelationLibrary` が依存 organ を補完し、不要になった依存 organ の痕跡器官化候補を判定する。
- 生存中 mutation は `OrganFoundation` の 10 秒 counter で動く。
- 生存中 mutation と相進化時の状態は checkpoint として保存される。
- 世代更新時には最良 checkpoint から次世代 `AIComponentSet` を作り、`mutationChanceG` を通して manager へ渡す。
- DNA / genome への永続保存と crossover はまだ残っている。

## パス参照の注意

`_移植元資料` の本文には、整理前の旧パスが残っている。履歴資料としては残すが、実装確認時は次のように読む。

- 旧 `Assets/script/Ingame/Creatures/Herbivore/...` は現 `Assets/script/Ingame/Creatures/before/Herbivore/...`
- 旧 `Assets/script/Ingame/Creatures/Predator/...` は現 `Assets/script/Ingame/Creatures/before/Predator/...`
- organ core は `Assets/script/Ingame/Creatures/after/organ/Core/...`

## 0. phase0_organ設計

目的は、旧 behaviour 本流から organ foundation 本流へ移行できる足場を固定すること。

### 実装済み

- `OrganFoundation` が `AnimalBrain.TickBrain()` を呼ぶ。
- 旧 `herbivoreBehaviour` / `predatorBehaviour` と二重移動・二重攻撃を起こさないための runner 境界を入れた。
- `OrganRelationLibrary` が依存 organ を自動導入する。
- `OrganPresetLibrary` が phase / species ごとの `AIComponentSet` preset を返す。
- `AnimalBrain` が inactive / vestigial organ を中央でスキップする。

### 残り

- 実機で移動、追跡、逃走、攻撃、魔法 action の二重実行がないか確認する。
- 旧 behaviour に残る HP / 死亡 / target / memory の正本移動時期を決める。

## 1. 支配種までの遺伝子定義

目的は、草食、捕食、上位捕食、支配種までで参照される genome 項目と保存場所を定義すること。

ここでは `ValueGene` と `AIComponentGene` の正本を決める。phase up 条件は `4.相の進化`、支配種としての設定・到達条件・追加付与 organ・役割・勝利接続は `5.支配種仕様・役割定義` に置く。

### 次にやること

- organ gene を既存 genome / DNA のどこに保持するか決める。
- magic aptitude を organ gene として扱う前提で、UI / DNA 表示対象を整理する。
- dominant 判定側が参照する phase / speciesID / organ / magic のデータ項目を確定する。

## 2. organ構成の遺伝

目的は、親から子へ organ 構成が継承される仕組みを作ること。

### 実装済み

- `AIComponentSet` を manager 経由で次世代 spawn へ渡す経路ができた。
- required organ と optional organ を gene 上で分けられる。
- 生存中 mutation で有利になった checkpoint も次世代候補に含まれる。

### 残り

- `AIComponentSet` を genome または DNA 表現へ保存する。
- crossover で `AIComponentGene` を継承する。
- preset と親由来 organ 差分の合成ルールを固める。

## 2.5. organ突然変異・生存中可塑性

目的は、organ 構成の変化を継承と世代更新から分離して、安全に検証できるようにすること。

### 実装済み

- 生存中 mutation は `mutationChanceT` と 10 秒 counter で抽選する。
- 世代更新時 mutation は `mutationChanceG` で抽選する。
- 失われた organ は即時削除せず、`isVestigialOrgan` / `VestigialOrgans` で保持する。
- vital organ は mutation 対象から除外する。
- mutation 後は依存 organ を再解決する。
- mutation event と generation log に organ 変化を残す。

### 残り

- mutation 発生頻度を実機で調整する。
- checkpoint score を「最後の状態」寄りにするか「最良状態」寄りにするか決める。
- component の実 Remove を解禁する条件を決める。

## 3. 世代更新と遺伝

目的は、評価、選抜、交叉、突然変異呼び出し、再配置、ログ、DNA 入出力をまとめること。

### 実装済み

- 生存中 mutation checkpoint を評価候補に含める。
- 最良 checkpoint から次世代 `AIComponentSet` を作る。
- 世代更新時 mutation を呼び、manager の `nextGenerationComponentSet` へ渡す。
- `GenerationLog` に organ checkpoint / active organ / vestigial organ / generation mutation summary を追加した。

### 残り

- DNA / genome serializer への organ gene 保存。
- 複数親 crossover と organ gene の混合。
- phase / magic / dominant 判定との統合。
- DNA 注入時の validation。

## 4. 相の進化

目的は、`predator -> highpredator` の phase up 条件と実行責務を固定し、`highpredator -> dominant` を魔法会得数による支配種化処理へ分離すること。

### 実装状態

- `predatorBehaviour.TryPhaseEvolution()` と `PredatorPhaseEvolutionAction` は、どちらも `predator -> highpredator` までに制限済み。
- `highpredator -> dominant` は相そのものの突然変異ではなく、`DominantAscensionAction` が通常属性魔法二種以上を条件に処理する。
- highpredator 化時に通常属性魔法を持っていない場合は、`MagicElementAffinityState` で一種を保証する。
- phase up 後の `AIComponentSet` は `OrganPresetLibrary` と `OrganFoundation` 経由で導入し、checkpoint に残す。

### 検証待ち

- 旧 behaviour 側と organ action 側の phase up 実装を、将来的にどちらかへ正本統合する。
- phase up 後の `Resource.resourceCategory`、`speciesID`、`AIComponentSet`、checkpoint、UI 表示にずれがないか実機で確認する。

## 5. 支配種仕様・役割定義

目的は、`dominant` になった個体を支配種として扱うための設定、到達条件、追加付与 organ、役割、表示、ログ、課題ステージ勝利条件との接続を固定すること。

### 完了 / MVP 実装済み

- 支配種到達条件は `category.highpredator` かつ通常属性魔法二種以上に固定済み。
- dominant 化時の追加付与 organ は `DominantAscensionAction`、`MagicElementAffinityState`、`SpaceMagicAction` を中心に実装済み。
- 空間魔法は支配種専用で、dominant 化時の自動導入でのみ獲得する。
- 魔法 organ は `isProtectedOrgan` により mutation / vestigial から保護する。
- `dominantCountPerGeneration` は `DominantLineageTracker` の自然到達数を上限にする。

### 検証待ち

- 実ゲーム内で highpredator が二種目の通常属性を得た瞬間に dominant 化するか確認する。
- 空間歪曲 / 空間転移の演出・効果量は、支配種仕様の次段階で調整する。
- 課題ステージの勝利条件へ dominant 指標を接続する作業は、ステージ側タスクで扱う。

## 問題なしと判断できる点

- Phase0 と mutation を分けたので、foundation は runner / install / checkpoint に集中できる。
- `2.organ構成の遺伝` と `2.5.organ突然変異・生存中可塑性` を分けたので、継承と変異を別々に検証できる。
- `3.世代更新と遺伝` は mutation ルール本体を持たず、呼び出しとログに集中できる。
- 生存中 mutation を checkpoint 化したので、魔法資質を生存中に得た個体が次世代へつなぐ機会を失わない。

## 残るリスク

- 生存中 mutation の即時 Remove は Unity の `Destroy()` タイミングと `AnimalBrain` の cache に注意が必要。初期実装では痕跡器官化と早期 return を優先する。
- checkpoint と genome 確定値の境界を UI / log に出さないと、観測時に混同しやすい。
- required / optional / vital の分類が曖昧なままだと、mutation で個体の基盤が壊れる。
- DNA / genome 永続化前は、再起動や外部保存をまたいだ organ gene 復元は保証できない。
