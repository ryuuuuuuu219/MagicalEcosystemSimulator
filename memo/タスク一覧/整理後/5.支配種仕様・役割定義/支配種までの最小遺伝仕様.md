# 支配種までの最小遺伝仕様

## 目的

草食、捕食、上位捕食、支配種までを同じ遺伝仕様で扱うため、支配種仕様側で必要になる最小データ仕様を固定する。

このメモは `5.支配種仕様・役割定義` の一部として扱う。phase up 条件は `4.相の進化` に置き、支配種になった後の到達条件、追加付与 organ、役割、勝利条件との接続をここから定義する。

## 最小構成

| 相 | 必須情報 | organ構成 |
| --- | --- | --- |
| 草食 | `ValueGene`、`HerbivoreGenome`、`Resource.resourceCategory = herbivore`、`speciesID` | herbivore preset |
| 捕食 | `ValueGene`、`PredatorGenome`、`Resource.resourceCategory = predator`、`speciesID` | predator preset |
| 上位捕食 | `ValueGene`、`PredatorGenome`、`Resource.resourceCategory = highpredator`、`speciesID` | highpredator preset |
| 支配種 | `ValueGene`、`PredatorGenome`、`Resource.resourceCategory = dominant`、`speciesID` | highpredator以上のpreset。専用organは未確定 |

## 到達条件の最小案

支配種として扱うための最小条件案を以下にする。

1. 個体の `Resource.resourceCategory` が `category.dominant` である。
2. 個体が有効な `speciesID` を持つ。
3. `AIComponentSet` に vital organ が揃っている。
4. 捕食系の主要 organ が active である。
5. highpredator 以上で必要な攻撃・魔法・mana系 organ のいずれかが active である。
6. 支配種仕様で定める追加付与 organ、またはそれに相当する役割が有効である。

## 追加付与organ

dominant 専用 organ は `5.支配種仕様・役割定義` で定義する。

保存場所は `AIComponentSet` と `ValueGene` に分ける。

理由:

- 現行 `OrganPresetLibrary.CreatePredatorPreset` は highpredator 以上を同じ強化条件で扱える。
- 支配種条件は phase rank 5 と既存 organ 構成から開始できる。
- 専用 organ を作る場合は、支配種の役割・演出・勝利条件・相進化と同時に定義した方が破綻しにくい。

ただし、後続で追加する場合の候補は以下。

- `DominantAuraEmitter`
- `SpeciesCommandDesire`
- `TerritoryControlAction`
- `HighOrderMagicAction`

## 旧genomeとの関係

- 支配種に到達しても、当面は `PredatorGenome` を使う。
- dominant 専用 genome は作らない。
- 支配種らしさは phase、speciesID、organ gene、ValueGene、mana条件、役割定義の組み合わせで表す。
- 旧 `attackDistance` などの廃止候補項目は、支配種判定には直接使わない。
