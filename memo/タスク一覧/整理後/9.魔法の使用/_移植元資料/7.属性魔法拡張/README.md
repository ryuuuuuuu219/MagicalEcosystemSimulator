# 7.属性魔法拡張

参照元: `git show f5a7322:memo/ROADMAP.md`

## 目的

- 群知性設計に先立ち、見た目と効果が結びついた属性行動を個体単位で導入する。

## 現状（git参照時点）

- 属性リスト構想のみ。
- 実装・評価式・行動接続は未着手。

## 作業区分

### アジャイル開発方針

- 1スプリントを「観測できる属性差分が1つ増える」単位にする。
- 本編シーンへ直結する前に、`Assets/Scenes/MaterialExperimentLab.unity` でマテリアル、発光色、攻撃軌跡、場への影響を小さく検証する。
- 各属性は「見た目」「行動効果」「場データ」「ログ/確認手段」を1セットで扱い、見た目だけ先行して仕様と乖離しないようにする。

### MVP

1. 火/氷/雷/風/空間の属性IDと表示色を定義する。
2. マテリアル実験場で、属性ごとの試験オブジェクトを横並びに確認できるようにする。
3. 1属性だけを選び、攻撃効果、クールダウン、可視化、ログまで縦に通す。
4. 実験場で確認できた差分のみを `Ingame.unity` へ移植する。

### スプリント候補

1. 実験場準備
- 新規シーンを作成し、カメラ、ライト、属性サンプル配置用ルート、測定用ルートを用意する。
- 既存の `FreeFlyCamera`、`WorldGenerator`、生成地形の `TerrainCollider` を実験場へ流用する。
- 完了条件: Unity で `MaterialExperimentLab.unity` を開き、属性サンプルの置き場が判別できる。

2. 属性データ最小実装
- 属性ID、表示色、相性、基本効果量を定義する。
- `MagicElement` enum を属性定義の入口にする。
- 実験用の飛翔体設定は `MagicMaterialExperimentLauncher` に集約し、`launchElement` 分岐で代入する。
- 非地形着弾時に対象を包む表現を使うか、従来の着弾点表現を使うかを Launcher 側の設定フラグで切り替える。
- 完了条件: 1つの参照元から属性名と色を取得できる。

3. 攻撃表現プロトタイプ
- 共通攻撃可視化へ属性色と軌跡差分を渡す。
- 先行実験として、カメラ左クリックで飛翔体を撃ち、接触地点で属性効果へ分岐させる。
- スペクテイター射撃の属性は Inspector の enum プルダウンで切り替える。
- 完了条件: 実験場で同じ攻撃でも属性ごとに見た目が変わる。

4. 行動・戦闘接続
- 個体の魔法適性、発動条件、クールダウンを戦闘判断へ接続する。
- 完了条件: 属性差がダメージ、移動阻害、範囲、追跡などの結果差として観測できる。

5. 場データ接続
- mana/heat/threat など既存の場データに属性効果の受け口を作る。
- 完了条件: 属性発動時に場の値または可視化が再現可能に変化する。
