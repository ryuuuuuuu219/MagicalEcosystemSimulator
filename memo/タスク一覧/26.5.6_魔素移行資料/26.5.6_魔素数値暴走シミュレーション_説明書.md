# 26.5.6 魔素数値暴走シミュレーション 説明書

## 目的

魔素経済で `mana` が条件次第で増え続けるかを、Unity を起動せずに確認するための簡易チェックツール。

対象ファイル:

- `memo/タスク一覧/26.5.6_魔素数値暴走シミュレーション.py`

## 前提

- 移動、待機、通常行動では `mana` を消費しない。
- 自然回復は考慮しない。
- 世代更新は考慮しない。
- 探索間隔の乱れは考慮しない。
- 草探索はシミュレーション上では固定値 `herbivore_eat_interval = 50秒` を基本にする。
- mana field 吸収間隔は `field_absorb_interval = 1秒` を基本にする。
- `mana` 上限は入れない。
- 上限の代わりに、必要な箇所だけ `y = log(1 + x) * scale` で変換して伸びを抑える。

## 実行方法

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py"
```

結果は標準出力に要約され、CSV が出力される。

既定出力:

```text
memo/タスク一覧/26.5.6_mana_simulation_result.csv
```

## よく使う実行例

魔法なし、近接と草だけ:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py"
```

魔法を 10 秒ごとに撃つ:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py" --magic-cooldown 10 --magic-damage 8 --magic-mana-cost 20
```

攻撃吸収を log 変換する:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py" --is-convert-attack-absorb --attack-absorb-log-scale 10
```

死亡時放散を log 変換する:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py" --is-convert-death-release --death-release-log-scale 10
```

field 吸収を入れる:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py" --field-initial-mana 1000 --mana-absorb-from-field-per-sec 0.5
```

phase 進化判定を見る:

```powershell
python "memo\タスク一覧\26.5.6_魔素数値暴走シミュレーション.py" --field-initial-mana 1000 --phase-up-mana-coefficient 0.00001 --phase-up-probability-cap 0.005
```

## 主要パラメータ

## 現コードから代入した既定値

2026-05-06 時点の `Assets/script` から拾った値を、シミュレーション既定値へ反映している。

### 個体数・初期 mana

出典: `ResourceDispenser`

- `grass_count = 100`
- `herbivore_count = 30`
- `predator_count = 10`
- `grass_initial_mana = 30`
- `herbivore_initial_mana = 100`
- `predator_initial_mana = 200`
- `mana_absorb_from_field_per_sec = 1`
- `field_absorb_interval = 1`
- `grass_eat_amount_per_touch = 30`
- `grass_spawn_cooldown = 10`

### 初期 HP

出典: `CreatureBalanceTuning`

- `herbivore_initial_hp = 35`
- `predator_initial_hp = 55`

草の HP は現コードに明確な HP 実装がないため、シミュレーション上の占位値として `grass_initial_hp = 1` を使う。

### 近接攻撃

出典: `CreatureBalanceTuning` / `predatorManager` / `PredatorCombatLibrary`

現コードの肉食攻撃は charge / bite / melee の 3 系統で、威力とコストは個体生成時に乱数で決まる。シミュレーションでは単一攻撃へ集約するため、代表値を使う。

- `attack_damage = 6`
  - biteDamage の範囲 `4-8` の中央値を代表値にした。
  - meleeDamage は `6-12`、charge は相対速度依存。
- `attack_interval = 1.35`
  - attackClock 範囲 `0.9-1.8` の中央値。
- `attack_mana_cost = 2.1`
  - biteManaCost 範囲 `1.2-3.0` の中央値。
  - chargeManaCost は `0.4-1.2`、meleeManaCost は `2.2-4.7`。
- `attack_hit_rate = 1.0`
  - 実装は距離・角度・接触・確率判定を持つ。
  - シミュレーションでは探索や空間条件を捨てるため、最大ケースとして 1.0 を既定にする。
- `damage_to_mana_rate = 1.0`
  - 実装では `bodyResource.AddMana(result.damage, out _)` により、与ダメージ量をそのまま mana 回復へ入れている。

### 死亡後放散

出典: `Resource` / `ResourceDispenser`

- `decomposeRate = 2` が現コードの死亡後 `mana` 減衰速度。
- シミュレーションでは新設計を優先し、`corpseMaxLifetime = sqrt(10 * deathMana)` と `manaDecayAfterDeathPerSec = deathMana / corpseMaxLifetime` を使う。

### mana field

出典: `ManaFieldManager`

- `gridResolution = 96`
- `diffusionRate = 1.5`
- `field_initial_mana = 0`
- `field_dissipation_rate = 0`

シミュレーションでは単一集約 field として扱うため、`field_diffusion_rate` は設定値として保持するが拡散形状は見ない。

### 魔法

現コードでは、魔法 projectile の速度・寿命・半径・field への影響に加えて、発射者 `Resource` が存在する場合の mana 消費、着弾ダメージ、魔法後の与ダメージ回収を接続済み。

そのため既定値は以下とする。

- `magic_cooldown = 0`
- `magic_damage = 12`
- `magic_mana_cost = 8`
- `magic_damage_to_mana_rate = 1.0`

魔法の暴走チェックをしたい場合は、`magic_cooldown` を 0 より大きくする。

## 現コードとの不一致・注意点

1. `mana` 上限なし設計と現コードのズレ
- 対応済み。
- `maxMana` はゲージ基準値であり、`AddMana()` は上限停止しない。

2. 草モデルのズレ
- 対応済み。
- 現コードも接触供給、消滅、`grassSpawnCooldown` 後の再生成モデル。

3. 死亡後放散速度のズレ
- 対応済み。
- 現コードも `sqrt(10 * deathMana)` から寿命と放散速度を導出する。

4. field 吸収は未実装
- 対応済み。
- 生存個体は 1 秒間隔で `ManaFieldManager` から吸収する。

5. 相進化判定は未実装
- 対応済み。
- 現コードでは肉食動物以上が 10 秒間隔で phase 昇格を判定する。

### 個体数

- `grass_count`: 草の総数
- `herbivore_count`: 草食の総数
- `predator_count`: 肉食の総数

### 初期値

- `grass_initial_hp`: 草の初期 HP
- `herbivore_initial_hp`: 草食の初期 HP
- `predator_initial_hp`: 肉食の初期 HP
- `grass_initial_mana`: 草の初期 mana
- `herbivore_initial_mana`: 草食の初期 mana
- `predator_initial_mana`: 肉食の初期 mana

### log 変換フラグ

上限を作らず、増加や放散だけを鈍らせるためのフラグ。

- `is_convert_attack_absorb`: 攻撃時吸収 mana を log 変換する
- `is_convert_magic_absorb`: 魔法時吸収 mana を log 変換する
- `is_convert_death_release`: 死亡時 field 放散 mana を log 変換する
- `is_convert_field_absorb`: field 吸収 mana を log 変換する

それぞれ `*_log_scale` で変換後の倍率を指定する。

式:

```text
converted = log(1 + raw) * scale
```

### 死亡と死骸

- `corpse_lifetime_scale`: 死骸寿命の係数
- `corpse_eat_rate_per_sec`: 肉食が死骸から吸う mana 量

死骸の最長残存時間:

```text
corpseMaxLifetime = sqrt(corpse_lifetime_scale * deathMana)
```

死亡後の放散速度:

```text
manaDecayAfterDeathPerSec = deathMana / corpseMaxLifetime
```

既定では `corpse_lifetime_scale = 10` なので、設計メモの `sqrt(10 * 死亡時mana)` と一致する。

### 草

- `grass_eat_amount_per_touch`: 接触 1 回あたりの供給量
- `grass_spawn_cooldown`: 食べられた草の再生成間隔
- `eat_rate_per_sec`: 草食が草から吸う速度
- `herbivore_eat_interval`: 草食イベント間隔。既定 50 秒

旧モデルの以下は使わない。

- `grassManaMax`
- `grassManaRegenPerSec`
- `grassEatCooldown`
- `grassDepletedRegenDelay`

### 肉食攻撃

- `attack_damage`: 近接攻撃ダメージ
- `attack_interval`: 攻撃間隔
- `attack_hit_rate`: 命中率
- `damage_to_mana_rate`: `1HP = +1mana` の係数
- `attack_mana_cost`: 攻撃ごとの消費 mana
- `same_target_drain_cooldown`: 同じ対象から連続回収できる間隔
- `corpse_detect_range`: 設定値として保持。現シミュレーションでは空間判定しない
- `prey_detect_range`: 設定値として保持。現シミュレーションでは空間判定しない

### 魔法

- `magic_cooldown`: 魔法の発動間隔
- `magic_hit_rate`: 命中率
- `magic_damage`: 魔法威力
- `magic_mana_cost`: 消費 mana
- `magic_damage_to_mana_rate`: 魔法後の与ダメージ回収係数
- `magic_recovery_window`: 設定値として保持。現シミュレーションでは集約処理
- `magic_target_count`: 範囲魔法などで巻き込む数
- `magic_max_net_gain_per_cast`: 1 発あたり黒字上限。負値なら無効
- `phase_magic_cost_multiplier`: phase ごとの消費補正

### mana field

- `field_initial_mana`: 初期 field mana
- `field_diffusion_rate`: 設定値として保持。現シミュレーションでは単一集約 field のため拡散形状は見ない
- `field_dissipation_rate`: field 自然減衰率
- `mana_absorb_from_field_per_sec`: 個体が field から吸う速度
- `field_absorb_limit_per_creature`: 個体ごとの吸収上限。負値なら無効
- `field_hotspot_attraction_weight`: 高濃度地点への誘引による吸収倍率
- `field_mana_radius_on_death`: 設定値として保持。現シミュレーションでは空間半径を直接使わない

### 突然変異・相の進化

世代更新は考慮しない。

突然変異・相の進化だけ、10 秒間隔で期待値として見る。

```text
probability = min(phase_up_probability_cap, phase_up_mana_coefficient * Δmana)
```

既定値:

- `phase_check_interval = 10`
- `phase_up_mana_coefficient = 0.00001`
- `phase_up_probability_cap = 0.005`

設計意図:

- `0.001% * Δmana`
- 上限 `0.5%`
- 10 秒間隔

ここでの `Δmana` は、簡易モデルでは `field_mana` を自身の座標グリッド値の代理として使う。

## CSV の見方

- `total_creature_mana`: 草食 + 肉食の合計 mana
- `field_mana`: mana field に残っている mana
- `corpse_mana`: 死骸に残っている mana
- `attack_gain`: 近接攻撃で得た mana 累計
- `magic_gain`: 魔法で得た mana 累計
- `grass_gain`: 草から得た mana 累計
- `field_gain`: field から吸った mana 累計
- `corpse_gain`: 死骸摂食で得た mana 累計
- `mana_spent`: 攻撃・魔法で消費した mana 累計
- `death_release`: 死亡・死骸減衰で field へ放散した mana 累計
- `expected_phase_ups`: phase 進化の期待回数

## 暴走判定の目安

まずは以下を見る。

- `total_creature_mana` が時間に対して単調に増え続けるか
- `field_mana` が吸収や減衰を超えて溜まり続けるか
- `attack_gain + magic_gain + grass_gain + field_gain + corpse_gain - mana_spent` が増え続けるか
- `expected_phase_ups` が短時間で過大にならないか

問題が出た場合の調整候補:

- 攻撃吸収に `--is-convert-attack-absorb` を使う
- 魔法吸収に `--is-convert-magic-absorb` を使う
- 死亡時放散に `--is-convert-death-release` を使う
- field 吸収に `--field-absorb-limit-per-creature` を設定する
- `magic_max_net_gain_per_cast` で魔法の黒字上限を設定する
