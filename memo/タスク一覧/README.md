# タスク一覧

`Assets/script` を基準に、未実装または本編未接続の作業だけを残す。

## 優先

1. 炭素経済廃止・魔素統合
- `Resource.carbon` / `maxCarbon` / `totalReleasedCarbon` と carbon 保存則を設計・実コードから削除する。
- `Resource` は削除せず、カテゴリ判定と個体側 `mana` 制御を担う既存コンポーネントとして維持する。
- `ResourceDispenser` は carbon 予算・carbon pool・carbon 初期割り当てを外し、スポーン設定、草リスト、世代環境リセットへ責務を縮退する。initiatespownerに改名する？
- 食物連鎖の「食べる理由」は残炭素量ではなく、与えたダメージ量を新規評価軸として維持する。
- 死亡地点に `mana` / `heat` を放出し、死骸は一定時間で消す。加えて死亡時の脅威度パルスを `threatmap_calc` へ放出する。
- `ManaFieldManager` の濃度に応じて、ヘルス回復、魔法使用解禁、魔法コスト判定を接続する。
- AI の意思判断に mana field の山への誘引を追加し、食料・脅威・境界・徘徊ベクトルと統合する。
- UI と世代評価から Carbon 表示・Carbon 評価軸を外し、与えたダメージ量の表示・評価軸を導入する。

2. 生態系コア強化
- `threatmap_calc` / `HeatFieldManager` / `ManaFieldManager` / `WindFieldManager` を行動判断へ本接続する。
- 地形、境界、遮蔽、記憶を移動評価に反映する。

3. 挙動AI分離
- `herbivoreBehaviour` / `predatorBehaviour` の感覚、欲求、移動、攻撃責務を分ける。
- manager から分離AIを配布・有効化する流れを作る。

4. 戦闘システム拡張
- `speciesID` / IFF / 敵対判定を戦闘に入れる。
- 待ち伏せ、追跡放棄、撤退の条件を再現可能にする。

5. 魔法の本編接続
- 実験場の `MagicElement`、属性弾、着弾効果、field 反映を `Ingame` の個体行動へ接続する。
- 魔法適性、発動条件、クールダウン、戦闘結果差をゲノムとAIへ入れる。

## 後続

6. 群知性
- 群れ意図、リーダー、役割分担、共有ターゲットを実装する。

7. 学習と進化
- 学習結果を世代更新とゲノム継承へ反映する。

8. 支配種
- 到達条件、遷移処理、上位個体の行動差分を実装する。

9. 課題ステージ
- クリア条件、敵対勢力、報酬、期間イベント、環境外乱を実行時に判定する。

## 保留メモ

- 戦術演出: 包囲、追い込み、囮、魔法連携の見せ方。
- 判断ログ: 追跡、放棄、撤退、魔法発動の理由ログ。
- 陣形: 多対多、側面展開、密集、散開、地形利用。
- 縄張り: 高知能向けの領域記憶と侵入状態。
- 熱量場: 検算ツール、線形以外の熱交換モデル比較。
