メモ上の矛盾と残課題

更新日: 2026-05-08

このファイルは、organ 実装へ切り替える前後で残る問題を整理する。


対応済み

1. 草食の進化方針

決定:
相の進化は肉食動物以上にのみ実装する。
organ の獲得、欠落、level 変化は草食動物も対象にする。
`PredatorPhaseEvolutionAction` による phase 昇格判定は肉食動物以上に限定する。

反映:
`memo/設定/設定7：遺伝システム.txt` に補足を追記済み。
`PredatorPhaseEvolutionAction` は `ManaFieldManager` を参照して predator -> highpredator -> dominant の phase 昇格判定を行う。


2. Zigzag 回避の扱い

決定:
Zigzag 回避 organ は完全削除方針。
現状実装に残る `ComputeEvasionVector()` のジグザグ処理は、organ 化しない。

反映:
`26.5.8_挙動AI分離_現状機能と設計方針.md` の切り出し候補から削除済み。


3. AI 管理方式

決定:
`node[]` ツリー案は旧案。
正本は `AIComponentGene` / `AIComponentSet` / `AnimalAIInstaller` による organ AddComponent 方式。

反映:
旧案だった `memo/設定/設定2：挙動AI管理.txt` は削除。
`memo/設定/README.md` からも参照を削除。


4. mana field の実装状態

決定:
初期設計時点では未実装。
現在は `ManaFieldManager` の基盤実装があり、個体吸収、魔法消費、相進化判定へ接続する段階。

反映:
`memo/設定/設計集約/field・環境場設計.md` に注記を追記済み。
`PredatorPhaseEvolutionAction` は mana field の値を相進化確率へ使う。


5. AnimalBrain と GroundMotor の接続

問題:
`AnimalBrain.TickBrain()` が移動ベクトルを作るだけで、実移動へ渡していなかった。

対応:
`AnimalBrain` が `GroundMotor` を保持し、`TickBrain()` の最後で `GroundMotor.Move()` を呼ぶようにした。
`GroundMotor.InheritVelocity()` を追加し、攻撃 organ が既存攻撃結果の速度を引き継げるようにした。


6. 記憶なし Desire の原点移動

問題:
`FoodDesire` / `PreyChaseDesire` / `ThreatAvoidanceDesire` が、記憶なしのとき `Vector3.zero` を目標にする可能性があった。

対応:
`FoodMemory` / `PreyMemory` / `ThreatMemory` に `hasMemory` を追加。
各 Desire は有効記憶がない場合 `AIMoveIntent.None()` を返すようにした。


7. CorpseEatAction が生体も食べる問題

問題:
grass 以外なら `Eating()` していたため、生きている個体から吸収する可能性があった。

対応:
`herbivoreBehaviour.IsDead` / `predatorBehaviour.IsDead` を見て、死体のみ食べるようにした。


8. ThreatPulseEmitter の攻撃時 API

問題:
存在パルスと攻撃パルスの API が分かれていなかった。

対応:
`ThreatPulseEmitter.EmitAttackPulse(Vector3 point)` を追加。
存在パルスは捕食者相の敵性判断装置として残す。
近接攻撃 organ の成功時に攻撃パルスを出す。


9. Sense organ の最低限の記憶更新

問題:
`FoodVisionSense` / `PredatorVisionSense` / `PreyVisionSense` / `ThreatVisionSense` がスタブで、Memory を更新しなかった。

対応:
既存 `herbivoreBehaviour` / `predatorBehaviour` の manager 参照を使い、候補リストから最小限の対象検出と Memory 更新を行うようにした。


10. 既存近接攻撃の organ 移植

問題:
`ChargeAttackAction` / `BiteAttackAction` / `MeleeAttackAction` が器だけで、既存の捕食攻撃を実行していなかった。

対応:
`PredatorCombatLibrary` に organ から呼べる公開ラッパーを追加。
`PredatorOrganCombatUtility` を追加し、`AIContext` / `PreyMemory` / 既存 behaviour から攻撃セットアップを作る。
`ChargeAttackAction` / `BiteAttackAction` / `MeleeAttackAction` は既存攻撃判定、ダメージ、mana 回収、mana cost、threat pulse、速度引き継ぎへ接続済み。


11. Magic organ の最低限の発射接続

問題:
魔法関連 organ は設計上の器のみで、既存の `MagicLaunchApi` と接続していなかった。

対応:
`MagicProjectileAttackAction` が `MagicLaunchApi.LaunchImmediate()` を呼ぶようにした。
`MagicAttackAction` は `MagicProjectileAttackAction` を確保して委譲する。
mana cost と cooldown 判定は organ 側から渡す。


12. 依存関係メモ

問題:
organ の自動導入に必要な依存関係が散らばっていた。

対応:
`organ_dependency_table.md` を追加し、Sense / Memory / Desire / Action / Magic の依存関係と Installer 側の必要処理を整理した。


残る問題

1. AnimalBrain の実行タイミング

`AnimalBrain.TickBrain()` は実装されたが、まだ既存 `herbivoreBehaviour` / `predatorBehaviour` の Update / FixedUpdate から呼ばれていない。

最終方針:
`herbivoreBehaviour` / `predatorBehaviour` は廃止し、organ による制御へ移行する。
そのため最終的には `AnimalBrain.Update()` または専用 runner の `FixedUpdate()` 内で `TickBrain()` を呼ぶ。

注意:
既存 behaviour と organ motor を同時に動かすと二重移動になる。
切り替え時は実行主体を一つにする。


2. Sense organ はまだ既存 AI と完全同等ではない

現在の Sense organ は最小実装で、以下は未反映。

- 視界首振り波形 `visionWaves`
- 草への経路安全判定 `IsGrassSafe()`
- TerrainCollider 遮蔽 Raycast
- Debug.DrawLine
- memorytime による記憶失効

必要:
`FoodVisionSense` の前提 organ として、`visionWaves`、`IsGrassSafe()`、遮蔽 Raycast、記憶失効を分離する。
`PredatorVisionSense` / `PreyVisionSense` / `ThreatVisionSense` も同じ基準で既存 AI と比較する。


3. AIMemoryStore と個別 Memory の責務が重複している

現在は以下が併存している。

- `AIMemoryStore`
- `FoodMemory`
- `PreyMemory`
- `ThreatMemory`

決定:
長期的には `AIMemoryStore` を正本にする。
短期的には個別 Memory を typed view として使う。


4. 未実装 Action organ

近接攻撃と魔法発射は移植済み。
以下はまだ `return false` または実効動作なし。

- `CounterAttackAction`
- `RandomEvasionAction`
- `MagicEvasionAction`
- `MagicDefenseAction`
- `RestDesire`
- `ReproductionDesire`

補足:
`CounterAttackAction` は既存スクリプト側に同等機能が見当たらないため、反撃条件から設計が必要。
`RandomEvasionAction` は zigzag 廃止方針と衝突しない形で、偶発回避として再定義する必要がある。
`MagicEvasionAction` / `MagicDefenseAction` は魔法被弾・防御条件の正本が未確定。


5. Combat フォルダの完全 organ 化

現状:
`PredatorCombatLibrary` は organ から呼ばれるようになったが、ファイル配置はまだ `Assets/script/Ingame/Combat` にある。
`AttackArcSettings` も同フォルダに残っている。

必要:
戦闘ロジックを `Assets/script/Ingame/Creatures/organ` 配下へ移すか、共通 combat library として残すか決める。
ラインレンダラーなどの攻撃可視化は Presentation 系 organ として整理する。


6. ThreatPulseEmitter の自動導入は未実装

方針:
捕食者相なら自動導入。
攻撃 organ がある場合も自動導入。

現状:
`AnimalAIInstaller` がまだ依存 organ 解決を実装していない。


7. organ 導入確率は資料のみ

`component_probability_table.md` に確率案はあるが、`AnimalAIInstaller` はまだ確率抽選を行わない。

必要:

- species / phase 別 default set
- random roll
- `AIComponentGene.enabled`
- `AIComponentGene.level`
- 自動導入 organ 解決


8. 既存 genome と organ gene の接続が未実装

現状の `HerbivoreGenome` / `PredatorGenome` は float 中心。
organ 方式では `AIComponentGene` / `AIComponentSet` を manager、世代交代、spawn へ接続する必要がある。


9. isDead の正本が旧 behaviour に残っている

現状:
`CorpseEatAction` は `herbivoreBehaviour.IsDead` / `predatorBehaviour.IsDead` を参照している。

必要:
`isDead` または生命状態を `AnimalBrain` / `AIContext` / lifecycle organ 側へ移し、相を追加しても同じ API で参照できるようにする。


10. CreatureRelationResolver は enum 順序依存のまま

決定:
`dominant` までしか追加しない予定なので、当面は問題として扱わない。

残留リスク:
category enum に途中追加が入る場合は見直しが必要。


11. 草食 organ 進化の評価軸が未定

方針:
草食も organ 進化対象。
ただし phase 進化はしない。

未決:
草食 organ の生存適応をどのスコアで選別するか。

候補:

- 生存時間
- mana 維持量
- 捕食回避成功
- 草摂食回数
- 世代内での平均距離移動
