---
全体像

- 現行の個体ゲノムは `HerbivoreGenome` と `PredatorGenome` の2系統。
- `WaveGene` は草食・捕食の両方で使う共有の波形遺伝子。
- 捕食のみ、攻撃範囲設定として `AttackArcSettings` をゲノム内に持つ。
- `maxHealth` / `health` / `maxEnergy` / `energy` は Behaviour 側の状態値であり、現行ゲノム項目ではない。
- Manager は `genome` と `nextGenerationGenome` を持ち、`useManagerGenome` / `useNextGenerationGenome` によってスポーン時の参照元を切り替える。
- ゲノムが未設定または不正な場合は、各 Manager の `ValidateOrRandomize` でランダム生成・補正される。

---
共有構造

`WaveGene`
- frequency
- amplitude
- phase

用途:
- `visionWaves`: 視覚・探索系の周期的な揺らぎ。
- `wanderWaves`: 徘徊移動の周期的な揺らぎ。

備考:
- 世代継承では配列要素ごとに交叉・突然変異される。
- 要素数が異なる場合は、親配列の末尾要素を補完的に使いながら最大要素数へ揃える。

`AttackArcSettings`
- radius
- arcDegrees
- length
- startOffset
- localDirection

用途:
- 捕食の `chargeArc` / `biteArc` / `meleeArc` に使用する。
- `Vector3` を含むため、世代継承では `BlendVector3` / `MutateVector3` 系で処理される。

初期化・補正:
- `herbivoreManager.ValidateOrRandomize` が未設定値を検出してランダム生成する。

DNA形式:
- `GenomeSerializer.EncodeGenome` で `HG:` + Base64 文字列に変換する。
- Base64の中身は `BinaryWriter` によるバイナリ形式。
- 現行バージョンは `2`。
- version 1 のDNAは読み込み時に `foodWeight` 以降の行動重みへ既定値を補完する。

---
世代継承処理

担当:
- `AdvanceGenerationController`

親選択:
- 通常は候補をスコア順に扱い、上位または保存済みDNAプールから親を選ぶ流れ。
- UI側には保存済みDNA、入力DNA、世代フェーズ切替の導線がある。

交叉:
- `CrossoverMode.Assign`: 親A/Bのどちらかを項目ごとに採用。
- `CrossoverMode.Average`: 親A/Bの平均値。
- `CrossoverMode.Interpolate`: 親A/Bの間をランダム補間。

突然変異:
- `enableMutation` が有効な場合のみ実行。
- float は共通の `MutateFloat` で変異する。
- `WaveGene[]` は `MutateWaves`、`Vector3` は `MutateVector3`、`AttackArcSettings` は `MutateAttackArc` で処理する。
- 草食の `predictIntercept` は交叉対象だが、現行の突然変異処理では bool 専用変異がない。

保存:
- 草食DNAは `GenomeSerializer.EncodeGenome` で保存・表示される。
- 捕食DNAは `PGJ:` JSONとして内部処理される。
- `saveOutputGenome` により世代出力DNAの保存有無を切り替える。

---
現状の設計上の注意点

- 草食と捕食で似た意味の項目名が揃っていない。
  - 草食: `visionturnAngle`
  - 捕食: `visionTurnAngle`
- DNA形式が草食と捕食で分かれている。
  - 草食: `HG:` + Base64 binary
  - 捕食: `PGJ:` + JSON
- 捕食には旧攻撃パラメータと複合攻撃パラメータが併存している。
- `WaveGene` は共通だが、視覚波形と徘徊波形の意味づけがコード外でまだ明確に分かれていない。
- 群知性、魔法適性、支配種段階に直結する遺伝子は未実装。
- 体力・エネルギー上限は現行ゲノムではなく固定バランス値寄りの扱い。

---
設計変更時の検討候補

1. DNA形式の統一
- 草食・捕食のDNAを共通Serializerへ寄せる。
- version管理を種別ごとに明示する。

2. ゲノム項目名の正規化
- 視覚、移動、代謝、記憶、攻撃などのカテゴリ名と項目名を揃える。
- 既存シーンのSerialize済みデータへの影響を確認する。

3. 複合遺伝子の扱い
- `WaveGene` と `AttackArcSettings` を個別構造として残すか、より汎用的な遺伝子ノードへ寄せるか決める。

4. bool遺伝子の変異ルール
- `predictIntercept` などの bool 値に、反転確率を持たせるか決める。

5. 拡張遺伝子の接続順
- 先に基礎行動・知覚を整理する。
- 次に群れ判断、魔法適性、支配種条件を追加する。
- 最後にUIのGenome Viewer / Injectorで一覧表示・編集しやすい形式へ整える。
