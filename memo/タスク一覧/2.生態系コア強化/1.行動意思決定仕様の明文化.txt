## 決定済み事項
- 現在の移動スクリプトは `food / prey` / `threat` / `boundary` / `wander` の重み付き合成で動いている。
- 草食は近接回避 + `zigzag evasion` を持つ。
- 肉食は `proportional navigation` 補正つき追跡を持つ。
- 実移動は `AnimalAICommon.ApplyMovement()` で処理される。
- `ThreatMapsGenerator` は `Ingame.unity` に配置済みで、`threatmap_calc` に `AddThreatPulse` / `GetLowThreatDirection` / `EvaluateBestDirection` がある。
- `PredatorCombatLibrary` は攻撃時に threat pulse を発生させる。
- `HeatFieldManager` は heat grid、拡散、減衰、`SampleHeat`、デバッグ描画を持つ。
- 草食・肉食・死骸分解は heat を発生させる。
- `heat field` は `Qbody / Qenv` を内部表現とし、温度換算、熱交換、解熱、恒常性、移動評価、状態異常まで定義済み。
- `mana field` は `Mbody / Menv` を内部表現とし、吸収、消費、拡散、死骸分解、転移、移動評価まで定義済み。
- `memo/設定/設定1：field設計まとめ.txt` は要点のみを持ち、`heat` / `mana` の詳細は別の演算仕様メモを正とする構成に整理済み。

## 基本方針
- 詳細な状態機械は別メモを正とし、このファイルでは行動意思決定との接続に必要な最小限だけを保持する。
- 先に固定する基本状態は `探索 / 回避 / 追跡 / 戦闘 / 離脱` の 5 状態とする。
- `警戒 / 警告 / 許可 / 認識` は IFF 導入後の拡張状態として扱う。
- `threat map` は移動欲求ベクトルの補助入力であり、状態遷移の単独決定因にはしない。
- 強い脅威、`heat`、境界圧、近接危険があるときは `回避` を優先する。
- 有効な `food / prey` 候補があり、かつ脅威が低いときに `追跡` を許可する。
- 攻撃条件が成立したときに `戦闘` へ移る。
- 交戦継続が不利なときは `離脱` を優先する。

## 状態一覧

| 状態 | 主目的 | 主な発火条件 | 主な停止条件 | 主参照要素 |
| --- | --- | --- | --- | --- |
| 探索 | 通常移動と候補探索 | 脅威・`heat` が低く、有効な `food / prey` 誘引が弱い | `food / prey` 誘引増大、脅威増大、`heat` 増大 | `wander`, `boundary`, `food/prey` 候補 |
| 回避 | 危険源から離れる | `threatWeight` 上昇、`evasionW > 0`、`heat` 上昇、境界圧 | 脅威・`heat`・境界圧の低下、近接回避終了 | `threat`, `threat map`, `heat`, `boundary` |
| 追跡 | `food / prey` へ接近 | 有効ターゲットあり、誘引重み十分、脅威が低い | ターゲット消失、記憶失効、脅威増大、戦闘移行 | `food / prey`, `memory`, `threat`, `heat` |
| 戦闘 | 攻撃成立の試行 | 攻撃距離・角度・クールダウン条件成立 | 対象死亡、対象喪失、エネルギー不足、離脱移行 | `target`, `attack cooldown`, `energy` |
| 離脱 | 交戦・接近の打ち切り | 追跡 / 戦闘継続が不利、脅威や `heat` が高い | 安全域への復帰、再追跡条件不成立 | `threat`, `heat`, `boundary`, `energy` |

## 状態ごとの判断要点

### 探索
- 通常移動の基底状態。
- `wander` を主軸にしつつ、弱い `food / prey` 誘引を受け取る。
- `threat` や `heat` が増した場合は `回避` へ遷移する。

### 回避
- 最優先の安全確保状態。
- `threat map` はここで強く使う。
- `food / prey` や `wander` は抑制し、危険回避ベクトルを優先する。

### 追跡
- `food / prey` 候補への接近状態。
- ターゲットの有効性、記憶、誘引強度を確認しながら継続する。
- 脅威や `heat` が増した場合は `回避` または `離脱` に遷移する。

### 戦闘
- 攻撃条件成立後の近接・攻撃試行状態。
- クールダウン、射程、角度、対象有効性を参照する。
- 不利な状況では `離脱` へ切り替える。

### 離脱
- 戦闘や過剰接近を打ち切るための状態。
- 安全域への退避を優先し、再追跡はすぐには許可しない。
- `threat` / `heat` / `boundary` / `energy` を複合的に見る。

## ベクトル合成ルール
- ベースは `food / prey` / `threat` / `boundary` / `wander` の重み付き合成。
- `threat map` は主に `回避` と経路選好の補助入力として使う。
- `heat field` は最低限「危険温度からの回避」には使う。
- `heat field` を「適温への誘引」にも使うかは未確定。
- `food / prey` 誘引は `回避` 優勢時には抑制する。
- `wander` は他の強い目的がないときの埋め草であり、優先度は低い。
- `boundary` は常時参照する安全制約として扱う。

## IFF / 警戒状態メモ
- このフォルダでは基本5状態との接続方針までに留め、実装本体は `4.戦闘システム拡張` 側で扱う。
- IFF 導入後に `認識 / 許可 / 警戒 / 警告 / 戦闘` の拡張状態を追加する。
- 判定には相手種別、種族 ID、攻撃能力の有無、敵対トリガーなどを使う。
- いったん警告した対象が離脱した場合は `警戒` に戻す。
- 接近継続や攻撃行動がある場合は `戦闘` に移す。
- 現段階では、まず 5 状態の基本遷移を先に固める。

## 初期縄張りルール
- このフォルダでは移動評価への入力候補として扱い、敵対判定・警告・戦闘化は後続タスクへ逃がす。
- 草原中心の初期領域を基礎縄張りとして扱う。
- 縄張り侵入をターゲット候補化する条件を別途定義する。
- 現状の「一定距離内の個体を敵性判断するのみ」から、どこまで 1 段階目で置き換えるかを決める必要がある。

## ログ観測方針
- 初期段階では `Debug.Log` ベースで十分。
- まず固定すべき観測値は以下。
- 現在状態
- 状態遷移理由
- 主参照ターゲット
- `threat` / `heat` / `boundary` の主要スコア
- 最終的にはラインレンダラーやテキスト表示による可視化も検討するが、優先度は低い。

## 次にこのファイルで詰めること

- 実装済み `threatmap_calc` のどの API を `ComputeTotalVector()` から参照するか固定する。

- 各状態でどのベクトルを参照し、どれを抑制するかを明文化する。
- ログ出力に必要な最小観測値を確定する。
