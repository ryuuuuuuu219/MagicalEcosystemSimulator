熱量の演算シミュレーション

目的：
個体の体温と環境の heat field を、同じ熱量系で扱えるように整理する。
この文書ではまず「内部表現」と「更新式」を定め、そのうえで UI 用の温度表示へ変換する。


◇前提

- 内部表現は「絶対熱量 Q」で統一する
- 個体体温も heat field も、内部的には温度ではなく熱量として持つ
- 行動判断や状態異常のしきい値は、必要に応じて熱量から温度へ換算して扱う
- 単位はひとまずゲーム内熱量単位として扱う
  現実の kJ や W に完全一致させることは目的にしない
  ただし係数の目安づけには人体スケールを参考にする


◇用語定義

項目：個体熱量 `Qbody`
概要：個体が内部に保持している熱量。

項目：環境熱量 `Qenv`
概要：heat field の各セルが保持している熱量。

項目：体温 `Tbody`
概要：`Qbody` を温度表示へ換算した値。

項目：環境温度 `Tenv`
概要：`Qenv` を温度表示へ換算した値。

項目：熱容量 `Cbody`
概要：個体熱量を温度へ変換するための係数。
個体ごとに `weight * 3.5` で算出する。
初期目安：70kg 個体なら `245`
根拠メモ：人体基準で 3.5 kJ/kg/K

項目：大気中熱容量 `Cenv`
概要：環境熱量を温度へ変換するための係数。
`Qenv` 専用の換算係数として扱う。
具体値は別変数として持つ。

項目：基準体温 `Ttarget`
概要：個体が最も安定しやすい目標温度。
叩き台：35℃

項目：公差
概要：`Ttarget` との差を快適域として無視できる範囲。
用途：移動判断用。

項目：許容範囲
概要：生存上問題のない体温帯。
用途：熱ダメージ・低温鈍足などの状態異常判定用。


◇温度変換

熱量から温度へ変換する。

```
T = Tbaseline + Q / C
```

- `T`：表示用温度
- `Tbaseline`：基準環境温度
- `Q`：熱量
- `C`：熱容量

UI 用の基準値：

- `Tbaseline = 25`
- `Cbody = weight * 3.5`
- 初期個体の目安は `weight = 70`、`Cbody = 245`
- `Cenv` は大気中熱容量として別変数で持つ

表示の叩き台：

```csharp
float Tbody = 25f + Qbody / Cbody;
float Tenv = 25f + Qenv / Cenv;
```

補足：

- これは UI としきい値判定のための換算である
- 内部更新ではなるべく `Qbody` と `Qenv` を直接扱う
- 体温 35℃ を目標にするなら、基準環境 25℃ に対して `Qbody = (35 - 25) * Cbody` を初期目安とする
- 初期個体が 70kg なら `Qbody = (35 - 25) * 245 = 2450`


◇個体熱量の更新

個体熱量は以下の和で更新する。

```
Qbody_next = Qbody
           + Qmetabolism
           + Qmovement
           + QexchangeFromEnv
           + Qexternal
```

- `Qmetabolism`：代謝で増える内部熱量
- `Qmovement`：運動で増える内部熱量
- `QexchangeFromEnv`：環境との熱交換
- `Qexternal`：火属性・氷属性など外部要因


◇項目別の整理

項目：代謝
概要：炭素の変換時に個体内部へ加える熱量。

現状：
- `carbonToEnergyRate = 0.5`
- `metabolicEnergyPerCarbon = 1.0`
- `metabolicHeatPerCarbon = 0.5`

現状の意味：
- 毎秒最大 `0.5 carbon` を消費する
- `energy` を `1.0 / carbon` で増やす
- 同時に `heat` を `0.5 / carbon` 相当で発生させている

整備方針：
- 「代謝熱をそのまま場へ撒く」ではなく、まず `Qbody` に加える
- その後、環境との熱交換で `Qenv` へ移る
- これで「体温」という中間状態を持てる

予定の叩き台：
- `carbonToEnergyRate = 1`
- `metabolicHeatPerCarbon = 0.1`


項目：運動
概要：移動や旋回に伴って個体内部へ加える熱量。

現状の energy cost：

```csharp
cost =
  (0.05 + 0.2 * moveDemand) * Time.fixedDeltaTime
  + 0.03 * accelerationDemand
  + 0.02 * brakingDemand
  + 0.0005 * turnDemand;
```

現状：
- これらは `energy` から減るだけで、まだ heat に戻していない

整備方針：
- 運動コストの一部を `Qmovement` として `Qbody` に戻す
- これにより、移動が多い個体ほど自己発熱しやすくなる

予定の叩き台：
- `HeatPerEnergyCost = 0.75`
- `Qmovement = energyCost * HeatPerEnergyCost`


項目：環境との熱交換
概要：個体熱量と、その個体がいるセルの環境熱量の差を平坦化する。

基本式：

```
Qexchange = k * (Tbody - Tenv) * dt
```

- `k`：熱交換係数
- `dt`：時間刻み

更新：

```
Qbody_next -= Qexchange
Qenv_next  += Qexchange
```

叩き台：
- `k = 0.04`

補足：
- `Tbody > Tenv` なら個体から環境へ熱が流れる
- `Tbody < Tenv` なら環境から個体へ熱が流れる
- 実装では、温度に変換せず `Q` 空間で直接書けるように係数を吸収してもよい

モデル候補：
- 線形：安定、基準モデル
- 4乗：高温差で急激に効く
- `tanh`：飽和しやすく断熱っぽい
- 閾値：恒温制御向け
- 速度依存：運動中だけ熱交換増加
- 面積依存：進化パラメータ化しやすい

現時点の推奨：
- まずは線形モデルで固定する
- 4乗、`tanh`、閾値、速度依存、面積依存は X-7-2 の低優先検討項目として切り分ける
- 熱交換係数 `k` は、将来の地形フィールド参照口だけ先に用意し、初期値は全地点同一で運用する


項目：外部属性
概要：火属性や氷属性の攻撃・地形効果など。

整備方針：
- 火属性は `Qexternal > 0`
- 氷属性は `Qexternal < 0`
- いずれもまず `Qbody` に加算し、その結果が許容範囲を外れたら状態異常に入る


◇heat field の更新

heat field は各セルの `Qenv` を持つ。

更新の流れ：

1. 個体からの熱交換を受け取る
2. 分解熱を受け取る
3. 恒星熱を受け取る
4. セル間で拡散する
5. 必要なら全体減衰する


項目：拡散
概要：各セルが近傍平均との差へならされる。

現状：
- `gridResolution = 96`
- `diffusionRate = 2.5`
- `decayRate = 0.08`
- 毎フレーム、上下左右の 4 近傍平均との差へ拡散
- その後 `1 - 0.08 * dt` で減衰

補足：
- 現状の減衰は「宇宙への放熱」「地表からの散逸」などをまとめた近似とみなす
- まずはこれでよいが、恒星熱を入れるなら減衰率との釣り合いを見る必要がある


項目：分解熱
概要：死骸の炭素を環境へ返す際に heat field へ加える熱量。

現状：
- `decomposeRate = 2.0`
- `decompositionHeatPerCarbon = 1.0`
- 半径 `3f` に散布

現状の意味：
- 毎秒 `2 carbon` 分解
- 返却量に対して `1.0 / carbon` の熱を場へ加える

整備方針：
- 分解熱は `Qenv` に直接加える
- 代謝熱や運動熱と違い、死骸には「保持体温」を持たせない前提でよい


項目：恒星熱
概要：天球上で恒星が公転し、高度に応じて heat field に入熱する。

現状：
- 未実装

設計の叩き台：

```csharp
float intensity = Mathf.Max(0f, Vector3.Dot(-sun.transform.forward, Vector3.up));
```

- `intensity` を 0..1 の日射係数として扱う
- 各セルに `intensity * solarPower * dt` を加える

叩き台パラメータ：
- `solarPower = 5`

初期方針：
- 地形影・緯度・季節・減衰率との釣り合いは X-7-1 の検算ツールで評価する
- 初期実装では全セル一様入熱で開始する


◇移動判断への接続

「暑い場所を避ける」ではなく、
「そのセルに移動したとき、自分の体温が基準体温へ戻りやすいか」で判定する。

最小構成の判定案：

1. 各候補セルの `Qenv` を読む
2. 仮の `Qexchange` を計算する
3. `Qbody_next_candidate` を求める
4. `Ttarget` との差が小さい候補を高評価にする

これにより、
- 寒い個体は暖かいセルを選びやすい
- 暑い個体は冷たいセルを選びやすい
- 同じセルでも個体状態によって評価が変わる


◇状態異常への接続

1. `Qbody` を温度へ換算する
2. `Ttarget` と比較して快適域かどうかを判定する
3. 許容範囲を超えたら状態異常に入る

例：
- `Tbody > upperLimit` で熱ダメージ
- `Tbody < lowerLimit` で低温鈍足

補足：
- 公差は「移動誘導用」
- 許容範囲は「生存判定用」
- ここは分けて保持した方がゲーム的に調整しやすい


◇現状からの移行順

1. 既存の heat field 拡散・減衰はそのまま使う
2. 代謝熱の投入先を「場」から「個体内部」へ変更する
3. `Qexchange` で個体と場をつなぐ
4. 運動熱を `Qbody` へ戻す
5. その後に恒星熱を追加する
6. 最後に移動 AI を「基準体温へ戻りやすいセル選好」へ接続する


◇決定事項

- `Qbody` と `Qenv` に同じ換算係数 `245` を使うか、別係数にするか
→Qenvのみ　この変数は「大気中熱容量」とする
- 近傍は上下左右だけで十分か、8近傍か
→4方向で
- 水場・日陰・地形高度で熱交換係数 `k` を変えるか
→フィールドは作っておいて同じ数値を使う
- 体格差を進化要素として熱容量へ反映するか
→resource に体重(`weight`)変数を追加　これと比熱（3.5 kJ/kg/K固定）で熱容量を算出する
- 恒星熱と減衰率のバランスをどう取るか
→X-7-1で解決を図る

◇結論

- 体温と heat field はどちらも熱量 `Q` で保持する
- `Qbody` は個体熱容量 `Cbody`、`Qenv` は大気中熱容量 `Cenv` で温度換算する
- 個体熱容量 `Cbody` は `weight * 3.5` で個体ごとに決める
- 代謝熱と運動熱はまず個体内部へ入れる
- 個体と環境の間は `Qexchange` で接続する
- 拡散近傍は 4 近傍、熱交換係数 `k` は当面全地点同値とする
- 分解熱と恒星熱は environment 側へ直接入れる
- UI としきい値判定だけ温度へ換算する
