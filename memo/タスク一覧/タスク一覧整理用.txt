タスク一覧整理用

目的:
既存のタスク一覧を、実装本線と「やりたいこと」の別軸に分けて整理する。
今回の前提は、群知性や多対多戦闘より先に魔法関連システムを設計すること。
旧 1.戦闘可視化強化 と旧 2.UI・プロパティ機能整理は統合し、1.戦闘可視化強化 にまとめる。

---
基本方針

0. 遺伝子設計変更
- 現行ゲノム構造、DNA形式、世代継承処理を整理する。

1. 戦闘可視化強化
- 場、場の UI、カメラ、UI ボタン追加をまとめる。
- 戦闘観測に必要な UI とカメラ導線を先に整える。
- ログ表示は X-2 へ退避する。

2. 以降の本線タスクは依存順に置く
- 2.生態系コア強化
- 3.挙動AI分離
- 4.戦闘システム拡張
- 5.魔素基盤構築
- 6.属性魔法拡張
- 7.群知性導入
- 8.学習と進化
- 9.支配種到達条件
- 10.課題ステージ
- 11.課題ステージ（期間イベント_環境外乱つき）

3. 「やりたいこと」は本線から独立した別軸とする
- 命名は `X-n.名称`
- 横断テーマ、将来案、演出案、観測案を格納する。

---
本線タスクの整理

1.戦闘可視化強化
- `場`
  - 魔力場、属性場、vector field、tensor field の実装単位を整理する。
- `場のUIの追加`
  - 場の表示切替、値・勾配・方向の表示を扱う。
- `カメラ`
  - 指定個体と目標個体を同時に見せる追跡カメラを扱う。
- `UIボタン追加`
  - 現行 UI の整理と、戦闘可視化に必要なボタン追加予定を扱う。

2.生態系コア強化
- 縄張り / 拠点ベース巡回の追加
- 座標好感度（場所記憶）評価の導入
- 高所・見晴らし評価の行動重み化
- threat map / heat field の行動意思決定統合
- 森と視界隠蔽の導入

3.挙動AI分離
- 制御AIに直書きされた機能の責務分離
- 感覚（視覚/聴覚/記憶連携）・欲求・攻撃のモジュール化
- AddComponent 運用を前提にした機能境界の整理

4.戦闘システム拡張
- 肉食の待ち伏せ条件追加
- 追跡放棄条件の精緻化
- 肉食動物→肉食動物戦闘の正式化
- 種族ID・IFF実装
- 高脅威判定、撤退条件、個体戦闘の状態遷移整理

5.魔素基盤構築
- 魔素グリッドの定義
- 拡散・吸収・消費の更新式実装
- 濃度差による能力補正実装
- 生体行動と魔素場の相互作用定義

6.属性魔法拡張
- 属性体系（火 / 氷 / 雷 / 風 / 空間）の効果定義
- 属性発動条件とクールダウン設計
- 戦闘 / 移動 / 環境との相互作用実装
- 観測 UI で属性発動の差分可視化

7.群知性導入
- GroupIntent データ構造定義
- 指揮者個体（リーダー）選出ロジック
- 追従 / 包囲 / 離脱判断の共有規則
- 魔法手段を含む多対一 / 多対多戦闘の判断基準整理
