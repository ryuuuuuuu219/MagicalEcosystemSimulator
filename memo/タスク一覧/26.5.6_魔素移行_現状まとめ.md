# 26.5.6 魔素移行 現状まとめ

作成日: 2026-05-06

## 現在の実装状態

- `Assets/script` 内の `carbon` / `energy` 参照は撤去済み。
- `Resource` は削除せず、個体・草・死骸の `mana` を持つコンポーネントとして維持している。
- `Resource.carbon` / `maxCarbon` / `totalReleasedCarbon` は `mana` / `maxMana` / `totalReleasedMana` に置換済み。
- `ResourceDispenser` は `manaPerGrass` / `manaPerHerbivore` / `manaPerPredator` を配給する形に変更済み。
- `EvaluationAxis.Carbon` は enum の同じ位置で `Mana` に置換済み。
- 世代交代時の評価スコアは `Resource.mana` を見る。
- 草食の採食、捕食者の死骸摂食は `Resource.Eating()` 経由で `mana` を移す。
- 生存中の移動・代謝による `mana` 自然消費は削除済み。
- 死亡後は `Resource.Decompose()` が `mana` を減衰させ、減衰分を `ManaFieldManager` へ放出する。
- 捕食者の攻撃成功時は、与ダメージ分を `mana` 回復し、攻撃ごとの `manaCost` を差し引く。
- 仮想ゲージ、選択 UI、オブジェクト一覧の評価値は Mana 表示へ置換済み。

## 26.5.6_設計メモ→実装開始1 後の進捗

- `category` を相として `grass / herbivore / predator / highpredator / dominant` へ拡張済み。
- `AddMana()` は `maxMana` で停止しない。`maxMana` はゲージ表示基準として残す。
- `mana > maxMana` は仮想ゲージ色で超過表示する。
- 肉食動物は mana field 値を使って `predator -> highpredator -> dominant` へ phase 昇格する。
- phase 昇格時に `speciesID` を既存最大ID+1までの重み付き抽選で更新する。
- `speciesID` が一致する肉食系個体は Friendly とみなし、生存攻撃・摂食対象から外す。
- `Resource.manaLog` に、攻撃吸収、攻撃コスト、草接触、死骸摂食、field 吸収、死亡放散を記録する。
- 生存個体は 1 秒間隔で mana field から `manaAbsorbFromFieldPerSec` 分を吸収する。
- 死骸放散速度は `corpseMaxLifetime = sqrt(10 * deathMana)`、`manaDecayAfterDeathPerSec = deathMana / corpseMaxLifetime` で導出する。
- 草は接触時に `grassEatAmountPerTouch` を供給して消滅し、`grassSpawnCooldown` 後に `ResourceDispenser` が再生成する。
- 魔法発射時に caster の `Resource.mana` から `magicManaCost` を消費し、着弾時に `magicDamage` を与え、与ダメージを `magicDamageToManaRate` で mana 回収する。
- 世代リセット破棄では `Resource.MarkGenerationResetDisposal()` を呼び、通常死亡の field 放散と明示的に分離する。

1. Unity Inspector の serialized field 確認
- `carbonPerHerbivore` などの旧フィールド名を `manaPerHerbivore` に変えたため、シーン上の既存設定値が初期値へ戻っている可能性がある。
- Unity 上で `ResourceDispenser`、`VirtualGaugeManager`、`AdvanceGenerationController` の値を確認する。

2. 手配置済み `Resource` の旧保存値
- prefab や scene に保存された `carbon` / `maxCarbon` は `mana` / `maxMana` に自動移行されない可能性がある。
- 初期スポーン個体はコードで `InitMana()` されるため影響は小さいが、手配置オブジェクトは確認が必要。

2.5. `mana` 上限なし設計と現コードのズレ
- 対応済み。
- `maxMana` は上限ではなくゲージ基準値。
- `AddMana()` / `Eating()` / field 吸収は `maxMana` で停止しない。

3. `mana` 加算内訳の観測
- 対応済み。
- `Resource.manaLog` に主要な mana 増減を記録し、選択 UI で末尾を表示する。

4. mana field から個体への吸収
- 対応済み。
- 草食・肉食とも生存時に 1 秒間隔で field 吸収する。
- 死骸摂食時は `Resource.Eating()` で mana を移動する。

5. 草の無限摂食対策
- 対応済み。
- 草は接触供給後に消滅し、`ResourceDispenser` が再生成する。

6. 魔法との接続
- 対応済み。
- 近接攻撃と魔法の `mana` 消費・回収を接続済み。
- 黒字化は許容するため、`magicMaxNetGainPerCast` は 0 以下なら上限なしとして扱う。

7. 通常死亡と世代リセット破棄の API 分離
- 対応済み。
- 通常死亡は `Decompose()` で field 放散する。
- 世代リセット破棄は `MarkGenerationResetDisposal()` で mana を 0 にしてから `Destroy()` する。

8. 数値暴走チェック
