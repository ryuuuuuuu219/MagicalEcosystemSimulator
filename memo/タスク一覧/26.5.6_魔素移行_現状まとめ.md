# 26.5.6 魔素移行 現状まとめ

作成日: 2026-05-06

## 現在の実装状態

- `Assets/script` 内の `carbon` / `energy` 参照は撤去済み。
- `Resource` は削除せず、個体・草・死骸の `mana` を持つコンポーネントとして維持している。
- `Resource.carbon` / `maxCarbon` / `totalReleasedCarbon` は `mana` / `maxMana` / `totalReleasedMana` に置換済み。
- `ResourceDispenser` は `manaPerGrass` / `manaPerHerbivore` / `manaPerPredator` を配給する形に変更済み。
- `EvaluationAxis.Carbon` は enum の同じ位置で `Mana` に置換済み。
- 世代交代時の評価スコアは `Resource.mana` を見る。
- 草食の採食、捕食者の死骸摂食は `Resource.Eating()` 経由で `mana` を移す。
- 生存中の移動・代謝による `mana` 自然消費は削除済み。
- 死亡後は `Resource.Decompose()` が `mana` を減衰させ、減衰分を `ManaFieldManager` へ放出する。
- 捕食者の攻撃成功時は、与ダメージ分を `mana` 回復し、攻撃ごとの `manaCost` を差し引く。
- 仮想ゲージ、選択 UI、オブジェクト一覧の評価値は Mana 表示へ置換済み。

## まだ残っている問題

1. Unity Inspector の serialized field 確認
- `carbonPerHerbivore` などの旧フィールド名を `manaPerHerbivore` に変えたため、シーン上の既存設定値が初期値へ戻っている可能性がある。
- Unity 上で `ResourceDispenser`、`VirtualGaugeManager`、`AdvanceGenerationController` の値を確認する。

2. 手配置済み `Resource` の旧保存値
- prefab や scene に保存された `carbon` / `maxCarbon` は `mana` / `maxMana` に自動移行されない可能性がある。
- 初期スポーン個体はコードで `InitMana()` されるため影響は小さいが、手配置オブジェクトは確認が必要。

2.5. `mana` 上限なし設計と現コードのズレ
- 設計上は `mana` 上限を入れない。
- ただし現コードは `Resource.InitMana(amount, amount)` で `maxMana = 初期 mana` を設定している。
- `Resource.AddMana()` は `maxMana > 0` の場合に上限で止めるため、攻撃回復など `AddMana()` 経由の増加は設計と異なる。
- 方針に合わせるには `maxMana` を廃止するか、`AddMana()` の上限処理を `isConvert*` 系の log 変換へ置き換える。

3. `mana` 加算内訳の観測
- 現在は合計 `mana` のみ。
- 攻撃、採食、field 吸収、魔法成果の内訳ログや UI 表示は未実装。

string manalogを追加
各処理の最後にmanalog+="attack drain" + result.damage() + "cost" + manaCost + "\n"のように加算内訳を記録する

4. mana field から個体への吸収
- 死亡後に field へ放出する処理はある。
- 生存個体が `ManaFieldManager` から一定速度で吸収する処理は未実装。
死骸に摂食したときにmanaを移動するようにしたい

5. 草の無限摂食対策
- 草の `mana` は減るが、採食済み状態、再供給クールダウン、個体側吸収間隔は未実装。
- 草食が強くなりすぎる可能性がある。
草は草食動物に接触した際に一定量のmanaを与え、瞬時に消滅するオブジェクトとする
再供給はresoursedispenserから生成をやり直す
現コードには `grassManaRegenRate = 0.5` の旧再生モデルが残っているため、接触供給・消滅・再生成モデルへ置き換える。

6. 魔法との接続
- 近接攻撃の `mana` 回復は実装済み。
- 魔法コスト、魔法後ダメージ回収係数、黒字化防止は未実装。
黒字化は許容する

7. 通常死亡と世代リセット破棄の API 分離
- 世代リセットは `Destroy` 中心で大きな副作用は避けている。
- ただし、通常死亡処理と世代破棄を明示的に分ける API はまだない。

8. 数値暴走チェック
- `1HP = +1mana`、個体 HP 25、攻撃コストなどの初期イメージは実装に近い形へ入った。
- 実運用前に簡易シミュレーションで、攻撃頻度、魔法回収、field 吸収、草供給の増殖傾向を見る。
- 実コードの HP は `HerbivoreMaxHealth = 35`、`PredatorMaxHealth = 55`。
- シミュレーション説明書には実コードから代入した代表値と、現コードとの不一致を追記済み。
- 相進化は `0.001% * Δmana`、10 秒間隔、上限 `0.5%` として `memo/設定/設定3：相の定義.txt` へ反映済み。
