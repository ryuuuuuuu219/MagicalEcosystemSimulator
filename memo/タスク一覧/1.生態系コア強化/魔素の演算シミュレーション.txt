魔素の演算シミュレーション

目的：
個体の内部魔素と環境の mana field を、同じ魔素量系で扱えるように整理する。
この文書ではまず「内部表現」と「更新式」を定め、そのうえで能力補正や UI 参照へ接続する。


◇前提

- 内部表現は「魔素量 M」で統一する
- 個体内部魔素も mana field も、内部的には同じ魔素量として持つ
- mana field は自然増加・自然減衰しない
- mana field は場全体で平坦になるように拡散する
- 魔法使用で消費され、空間魔法や死骸分解で局所的に増加する


◇用語定義

項目：個体魔素 `Mbody`
概要：個体が内部に保持している使用可能魔素量。

項目：環境魔素 `Menv`
概要：mana field の各セルが保持している魔素量。

項目：魔素容量 `Mcapacity`
概要：個体が安定保持できる内部魔素量の上限。

項目：基準魔素帯 `Mtarget`
概要：個体が最も安定しやすい内部魔素量の目安。
容量上限そのものではなく、行動判断の基準値として使う。

項目：公差
概要：`Mtarget` との差を無視できる範囲。
用途：移動判断用。

項目：許容範囲
概要：内部魔素量として生存上問題のない範囲。
用途：魔素欠乏、魔素過飽和などの状態異常判定用。

項目：魔素吸収率 `manaAbsorbRate`
概要：環境から内部へ魔素を取り込む速度係数。

項目：魔素放出率 `manaReleaseRate`
概要：内部魔素を意図的に場へ戻す速度係数。
初期実装では未使用でもよいが、設計上は持っておく。


◇個体魔素の更新

個体魔素は以下の和で更新する。

```
Mbody_next = Mbody
           + MabsorbFromEnv
           - MspellCost
           - MreleaseToEnv
           + Mexternal
```

- `MabsorbFromEnv`：環境からの吸収
- `MspellCost`：魔法使用による消費
- `MreleaseToEnv`：意図的な放出
- `Mexternal`：外部イベントによる増減

補足：
- 初期実装では `MreleaseToEnv` は 0 固定でもよい
- `Mbody` は `0 .. Mcapacity` に clamp する前提で扱う


◇項目別の整理

項目：環境からの吸収
概要：個体がいるセルの `Menv` から `Mbody` へ魔素を移す。

基本式：

```
Mabsorb = manaAbsorbRate * absorbAffinity * f(Menv, Mbody) * dt
```

最小構成の叩き台：

```
Mabsorb = manaAbsorbRate * dt
```

ただし実際の移動量は以下で制限する。

```
Mabsorb = min(Mabsorb, Menv, Mcapacity - Mbody)
```

整備方針：
- 環境に十分な魔素があり、内部容量に余裕があるときだけ吸収する
- 魔法が使えない動物は `manaAbsorbRate = 0` としてよい
- 将来的には遺伝子や属性適性で `absorbAffinity` を変える


項目：魔法使用
概要：個体内部魔素を消費して魔法効果へ変換する。

基本式：

```
MspellCost = spellCost * spellOutputScale
```

整備方針：
- 消費はまず `Mbody` から差し引く
- 不足時は不発、弱発、反動のどれかに落とす
- 初期設計では「足りなければ不発」を基準とする


項目：外部イベント
概要：死骸分解や空間魔法の影響で、内部または環境魔素が増減する。

整備方針：
- 個体に直接付与するより、原則として `Menv` を介して扱う
- これにより、場としての分布変化を可視化しやすくする


◇mana field の更新

mana field は各セルの `Menv` を持つ。

更新の流れ：

1. 個体の吸収・放出を反映する
2. 死骸分解による魔素生成を反映する
3. 転移など空間魔法による魔素生成を反映する
4. セル間で拡散する

補足：
- 熱量場と違い、自然減衰はさせない
- 総量監視も初期段階では行わない
- 世代更新時に 0 初期化する


項目：拡散
概要：各セルが近傍平均との差へならされる。

基準式：

```
Δmana = diffusionRate * (neighborAverage - self)
mana_next = mana + Δmana
```

叩き台：

```
diffusionRate = dt / 300sec
```

初期方針：
- 近傍は 4 近傍（上下左右）で固定する
- グリッドは heat field と同解像度運用を第一候補とする
- ただし manager は別管理でよい


項目：死骸分解
概要：死骸の分解を空間魔法の一種として扱い、所属グリッドへ魔素を加算する。

基本式の叩き台：

```
Mdecompose = decomposeRate * manaPerDecompose * dt
```

整備方針：
- 死骸が属するセルへ直接加算する
- 熱量場の分解熱とは別系統で扱う
- 支配種進化や高魔素地帯形成の起点になりうる


項目：転移
概要：発動地点と転移先を結ぶ線分が通るグリッドへ魔素を分散加算する。

整備方針：
- 経路セル列を取得し、各セルへ均等または重み付きで配分する
- 発動距離が長いほど総発生魔素量を増やしてよい
- ただしそのぶん使用コストも増やす

コスト案：

```
teleportResistance = sum(Menv along path)
MspellCost = baseCost + teleportResistance * resistanceScale
```

補足：
- 「高魔素地帯を通る転移は重い」という世界観にできる
- 同時に、転移の痕跡として経路に魔素が残る


◇個体能力への接続

mana field は直接ダメージ源ではなく、能力補正源として扱う。

接続候補：
- 魔法使用可否
- 魔法威力補正
- 詠唱安定度補正
- 進化条件
- 地域適応

最小構成の判定案：

1. `Mbody` が必要魔素量を超えているか確認する
2. 現在セルの `Menv` が最低濃度を満たすか確認する
3. 両方満たすときだけ魔法行動を許可する


◇移動判断への接続

熱量場のような「安全域」ではなく、
「そのセルへ行くと今後必要な `Mbody` を確保しやすいか」で判定する。

最小構成の判定案：

1. 各候補セルの `Menv` を読む
2. 仮の `Mabsorb` を計算する
3. `Mbody_next_candidate` を求める
4. `Mtarget` または必要魔素量に近づく候補を高評価にする

これにより、
- 魔法を使いたい個体は高魔素側へ寄りやすい
- 魔法不要の個体は mana field を無視できる
- 地域差が進化と行動に接続しやすい


◇状態異常への接続

1. `Mbody` を許容範囲と比較する
2. 下限を割れば魔素欠乏、上限を超えれば過飽和とする
3. 状態異常は行動制限や暴発リスクへ接続する

例：
- `Mbody < lowerLimit` で魔法不能
- `Mbody > upperLimit` で制御不安定

補足：
- 公差は「移動誘導用」
- 許容範囲は「生存・行動制限判定用」


◇現状からの移行順

1. mana field グリッドと更新ループを追加する
2. 拡散式 `Δmana` を実装する
3. 死骸分解による局所魔素生成を追加する
4. 個体側に `Mbody`、`Mcapacity`、`manaAbsorbRate` を追加する
5. 吸収処理で `Menv -> Mbody` を接続する
6. 魔法コストと行動条件へ接続する
7. 最後に移動判断へ接続する


◇決定事項

- 内部表現は個体 `Mbody`、環境 `Menv` の魔素量で統一する
- mana field は自然増加・自然減衰しない
- 拡散近傍は 4 近傍（上下左右）で固定する
- 総量監視は行わず、世代更新時に 0 初期化する
- 死骸分解は空間魔法の一種として、その所属セルへ魔素を加算する
- 転移は経路セルへ魔素を分散加算し、コストは経路上魔素量総和を抵抗として扱う
- 魔法が使えない個体は `manaAbsorbRate = 0` とする

◇結論

- mana field は熱量場と同様に「個体内部量」と「環境場」を分けて扱う
- 個体は `Mbody`、環境は `Menv` を持ち、吸収と消費でつなぐ
- 場の変化は拡散、死骸分解、転移で作る
- 魔法行動と移動判断は「必要魔素量を確保できるか」で接続する
