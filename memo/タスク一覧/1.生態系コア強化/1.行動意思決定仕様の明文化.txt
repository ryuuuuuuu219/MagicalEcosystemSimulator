1.行動意思決定仕様の明文化

目的
- threat map / heat field / wander / 餌 / 脅威 / 境界の優先関係を明文化する。
- 実装時に herbivore / predator / 共通AIで解釈がぶれないようにする。

対象資料
- memo/タスク一覧/1.生態系コア強化/設定：動物の移動・場（map）のまとめ.txt
- memo/タスク一覧/1.生態系コア強化/設定：ゲノム構造（基礎拡張）.txt

作業項目
1. 既存仕様の抜け確認
- threat map の用途、heat field の用途、wander との競合条件を洗い出す。

2. 意思決定レイヤーの定義
- 探索、回避、追跡、離脱の状態をどう分けるか整理する。
- 状態遷移の発火条件と停止条件を文章化する。

3. ベクトル合成ルールの定義
- attraction / repulsion / boundary / terrain 補正の合成順を決める。
- threat 優勢時、heat 優勢時の打ち消し条件を決める。

4. ログ観測項目の定義
- 追跡開始、継続、放棄の判定理由をどの値で出すか決める。

5. IFF / 警戒状態の仕様化
- 未認識、認識、警戒、警告、許可、戦闘状態の遷移条件を定義する。
- 相、種族ID、派閥ID、反撃能力の有無でどう遷移が変わるかを整理する。
- 警告後に離脱なら警戒へ戻す、接近なら戦闘状態へ移すなどの分岐を明記する。

6. 初期縄張りルールの仕様化
- 草原を中心とした円状領域を基礎実装の縄張りとして定義する。
- 縄張り侵入をターゲット候補化する条件を整理する。
- 現状の「一定距離内の個体を適性判断するのみ」から、どこまでを 1 段階目で置き換えるか決める。

完了条件
- 行動状態と場データの関係が文章だけで追える。
- 実装側が参照する優先順位と停止条件が明記されている。
