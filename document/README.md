# document

このディレクトリは、まず `開発者向け` と `ユーザー向け` を分け、その中を `初回リリース` と `リリース後` で整理する。

## 構成

- `開発者向け/初回リリース`
  初回リリースに向けた実装前提、非対象範囲、導線仕様を置く。
- `開発者向け/リリース後`
  リリース後の拡張方針、段階的な実装テーマを置く。
- `ユーザー向け/初回リリース`
  初回リリース時点でユーザーに見せる画面仕様や利用導線を置く。
- `ユーザー向け/リリース後`
  初回リリース以降にユーザーへ案内したい拡張予定を置く。

## 方針

- 初回リリース側は最小要件に絞る。
- リリース後側は中長期の拡張を扱う。
- ユーザー向け文書では操作と見え方を優先する。
- 開発者向け文書では責務、実装境界、非対象を明記する。
